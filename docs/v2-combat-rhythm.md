# v2 戰鬥節奏設計指引(拍子總表 + 鐵則)

> **這份文件給誰**:未來在這個 repo 上加動詞/道具/調時序的 Claude(和使用者自己)。
> **一句話**:這個遊戲的戰鬥是節奏遊戲——每個動詞都是「預告拍→執行拍→恢復拍」,
> 加新東西之前先看這張總表,讓新拍子跟現有拍子**咬合**,不要各調各的。
> 判定時刻大多由 clip tags 自動導出(animation-workflow §4)——**在 punch-studio 擺影格=在編節奏**。

---

## 0. 為什麼節奏是這個遊戲的骨架(不是裝飾)

程式架構已經收斂成一台節奏機:所有動詞都走**排程動作**模式(按下=承諾起手,效果落在
impact/release 幀),反擊拳字面上就是節奏判定(擋下→停頓→窗內反擊),兩拍桶
(砸中→落地閃 1 秒→爆)是最新的實例。這不是巧合——**「按下不等於生效」給了對手讀拍子
的機會,攻防才成立**。

**節奏 vs 混亂(招牌張力)**:玩家的動詞守節奏(可讀、可跳舞);環境的事故不守節奏
(元素站、地板化學、野生桶)。喜劇=完美的一拍被事故打斷。所以:**加玩家動詞→查這份文件;
加環境事故→故意不對拍(隨機/不連續),但要有預警拍(如站的 3s 收縮環)**。

## 1. 三層節奏(改哪層,想哪層的事)

| 層 | 時間尺度 | 管什麼 | 住哪 |
|---|---|---|---|
| **微觀** | <0.2s | 打擊感:hitstop、impact 定格幀、per-limb lag follow-through | clip 資料 + fx(addHitstop)+ ANIM.blend |
| **中觀** | 0.2–2s | **攻防對話(設計主戰場)**:起手/命中/恢復、反應窗、引信、暈眩 | v2-state 常數(大多由 clip tags 導出) |
| **宏觀** | 5s+ | 場面張力曲線:站輪噴、補給重刷、三階段收容升級 | v2-state 常數 + v2s.*Cur(階段變速) |

## 3. 自然形成的「準拍」(調新值先試著落在這些格子上)

Audit 發現時序自然聚在幾個值附近——不是刻意設計,但**證明手感自己收斂出拍子**:

- **~0.28s(基礎脈搏)**:拳 impact(0.23–0.38)、rigged 手收招 0.3、AI_THROW_DELAY 0.3
- **~0.55s(半拍)**:PUSH_WIN、AI_GRAB_DELAY、AI_BACKOFF_T 三個**互不相干的常數都選了 0.55**、桶飛行 0.55、FUMBLE_T 0.5、丟後 punchCd 0.5
- **~1.0–1.2s(全拍)**:落地引信 1.0、揍桶引信 1.0、暈眩 1.2
- **~3s(樂句)**:站預警 3、推開冷卻 3、AI 掙脫 3.3

新常數先問:「它是脈搏、半拍、全拍還是樂句?」落格子=玩家能憑肌肉記憶預測;
真要落格子外(如 0.7)要有理由(階段升級的「變快」故意破格=製造緊張)。

## 4. 鐵則(加/改任何戰鬥動詞前過這四關)

1. **三拍結構**:每個動作要有「預告拍→執行拍→恢復拍」。沒有預告=對手不可反制(bug 級);
   沒有恢復=無腦連打(平衡級)。恢復拍=對手的懲罰窗,要捨得給。
2. **防守方永遠至少一拍**:任何攻擊從「可見預告」到「傷害落地」之間,防守方要有 ≥ 半拍
   (~0.55s)的反應機會——落地引信 1s、站預警 3s、拳起手 0.28s+格擋慢動作輔助,都是這條。
   **檢驗法**:算「逃離/反制所需時間」vs「窗口長度」(例:爆風 95px ÷ 速度 168px/s = 0.57s
   < 1.0s 引信 ✓ 緊但公平)。
3. **同鍵不同時機=不同結果**(深度來源):格擋鍵三層結果是範本;新動詞可考慮同樣結構
   (按住/點按、時機分層),**別加新按鍵**(按鍵極簡是既定方向)。
4. **出手=拍子承諾**:排程中被打斷=取消且不退資源(道具次數已扣不退=現況)。承諾感是
   節奏遊戲的代價結構,別加「可隨時取消」的動作。

## 5. 加新動詞/道具的節奏 checklist

- [ ] 預告拍多長?(填 `ITEM_SPEC.delay` = clip impact 幀÷60,studio 編動作時就決定)
- [ ] 恢復拍多長?(`ITEM_CAST_RECOVER` 夠嗎,還是要專屬值?)
- [ ] 防守方的一拍在哪?(算逃離時間 vs 窗口;§4-2 檢驗法)
- [ ] 落在哪個準拍格子?(§3;有意破格要寫理由)
- [ ] 跟現有拍子的咬合:會不會跟 PUSH_WIN/STUN_T/引信形成必中連段?(必中連段只允許
      「先暈再丟」這種**花了兩拍成本**換來的;一拍成本就必中=過強)
- [ ] 宏觀:階段升級要不要變快?(加進 `applyStage` 的 *Cur)
- [ ] 更新本文件 §2 總表

| 暈眩 1.2 vs AI 抓延遲 0.55 | 玩家被暈也有 ~0.65s 看提示 | AI 延遲歸零=「AI 太強」重演(已踩過) |
| 掙脫 3.3s vs 扛人走速 CARRY_SLOW 0.6×168≈100px/s | 扛 330px≈半場=丟或入艙要果斷 | 掙脫太快=抓白抓(45→30 已調過一次) |
| 揍桶引信 1.0(原地)vs 落地引信 1.0(被丟) | 兩者同拍=玩家學一次 | 改其一要想清楚為何不同 |

## 7. 調整入口速查

| 想調的拍 | 改哪 |
|---|---|
| 拳的預告/命中拍 | punch-studio 移 impact 幀 → 重貼 clip(STRIKE_DELAY 自動跟) |
| 丟桶/丟人時序 | clip 的 release/hold tag(自動跟)+ `BARREL_LOB.T`/`PERSON_LOB.T`(飛行拍) |
| 落地爆的反制窗 | `BARREL_LAND_FUSE` |
| 反應窗/冷卻類 | `PUSH_WIN`/`PUSH_CDT`/`COMBO_*`/`STUN_T`/`REGRAB_CD`(v2-state) |
| 打擊感(微觀) | clip 的 impact hold 幀數 + `addHitstop` 呼叫點的值 |
| 場面節拍器 | `STATION_INTERVAL`/`*_RESPAWN` + `applyStage` 的階段值 |
