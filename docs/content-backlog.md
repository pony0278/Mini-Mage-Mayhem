## 0. 鐵則：控制方案鎖定（手機友善）

> **永遠只有四個動作。變化來自 roguelike 的 build 選擇，不是來自加按鍵。**

| 動作 | 桌機 | 手機 | 變化來源 |
|---|---|---|---|
| 移動 | WASD | 左搖桿 | — |
| 主攻 | 左鍵 | 右側射擊鈕 | 四元素 + 融合 + 修飾符（已有）|
| 衝刺 | Space / Shift | 衝刺鈕 | **沾元素 + 升級**（移動流派）|
| **副攻** | 右鍵 | 一顆副攻鈕 | **本局 build 決定是哪個法術**（插槽）|

- **不准**再加第五個動作鍵。新法術一律塞進「副攻插槽候選池」或做成衝刺/主攻的修飾符。
- 手機瞄準另議（自動鎖定最近敵人 / 右搖桿 / 拖曳），不屬於本清單。

## 1. 兩大可改造軸（變化來源）

### 1A. 元素衝刺（移動流派，不佔按鍵）
衝刺沾上目前主攻元素，把走位變成佈場手段：
- 火衝：留火、點燃草地、**引爆經過的毒霧**（也會燒到自己）
- 冰衝：留冰面（自己會滑）、凍結經過的水池
- 毒衝：拉出毒霧尾巴
- 雷衝：短瞬移 + 落點電一下水池
- 升級池：多一段衝刺、CD↓、衝刺穿過敵人傷害/擊退、尾巴更大
- **重用**：衝刺 i-frame + 留痕系統。成本：低。

### 1B. 副攻插槽（一顆按鍵，build 決定內容）
這一局抽到什麼，副攻就是什麼。所有「想加的法術」都當成它的候選（見 §4 候選池）。

## 2. 複合組合（emergent combos）— 明確的設計目標

> 因為「衝刺元素 × 副攻法術 × 主攻元素 × 修飾符」彼此獨立，玩家會**自己兜出**沒被預設的爽 combo。這是這套設計的賣點，要刻意保留與鼓勵。

範例（也是 build 流派的雛形）：

| 組合 | 玩法 | 流派感 |
|---|---|---|
| **風衝刺 + 土拳頭** | 風衝突進拉近/把怪掀起 → 土拳把怪打進牆或你的災難區，並能打破薄牆改地形 | 突進打擊流 |
| 火衝刺 + 油副攻 | 鋪油 → 火衝穿過點燃，騎著一條爆炸尾巴 | 縱火犯流 |
| 黑洞副攻 + 毒主攻 + 火 | 黑洞把怪吸進毒霧 → 點火集體引爆 | 聚怪引爆流 |
| 風副攻(吹) + 任意災難 | 把自己造的火/毒吹到敵人臉上、或從自己身上吹開 | 災難搬運流 |
| 冰牆副攻 + 火主攻 | 冰牆擋路 → 火把牆融成蒸氣形成煙幕遮蔽 | 控場視野流 |

> 守則：新法術設計時，**優先考慮它和「衝刺元素 / 既有災難」能不能兜出 combo**，而不是它單獨多強。

## 3. 第一批（✅ 已實作）

> 已上線到 `index.html`（並同步進 `camera-sandbox.html`）。元素衝刺需先在升級拿到元素才有留痕；副攻預設冰牆，升級可換土牆。衝刺/副攻升級已進 `upgradePool`（疾風步、衝刺強化、副攻：土牆/冰牆）。

1. **元素衝刺**（§1A）— 0 新按鍵。衝刺沾當前元素留痕 + 撞擊敵人傷害/擊退。
2. **副攻插槽 + 第一顆法術：冰牆 / 土牆**（唯一新按鍵：右鍵 / Q）
   - 手機友善：按副攻 → 沿瞄準方向長出**固定長度**的牆（按住預覽、放開生成）；**非自由畫線**。
   - 冰牆：擋敵人/投射物、附近減速、跨水池凍住、**遇火融成蒸氣**、可冰爆碎裂。
   - 土牆：擋路、**被爆炸打破**（重用現有薄牆破壞）、可當掩體擋遠程小鬼。
   - **重用**：牆＝寫進 `game.map` 的臨時 tile（帶壽命）→ 碰撞、火融冰、爆炸破牆全部沿用既有 tile 邏輯。
   - 升級池：牆更長/更久、冰牆減速更強、土牆爆破時噴碎石傷害。
   - 成本：中。

> 第一批刻意涵蓋「移動軸 + 副攻軸」兩條可改造軸，先把框架立起來，之後加內容只是往候選池塞。

## 4. 副攻插槽候選池（往後持續追加）

> 全部共用同一顆副攻鍵，靠 build 切換。✅ = 已實作（升級池可裝備）。

- ✅ **油**：地上鋪油，遇火大範圍爆燃（會連鎖鄰格油）。佈場縱火流。`TILE_OIL` + `updateOil`。
- ✅ **風陣 / 陣風**：錐形擊退 + **把火/毒/蒸氣往瞄準方向吹**。災難搬運流。`windGust`。
- ✅ **土拳（改為「主攻」模式，非副攻）**：升級「土拳・肉搏」把**主攻擊**換成近戰拳（高傷擊退、破牆、附帶當前元素效果）→ 肉搏戰士流派。`meleePunch` + `stats.mainMode`。
- 副攻**預設為「無」**，靠升級裝備（冰牆/土牆/油/風/黑洞）。
- ✅ **黑洞**：吸聚敵人 + 災難雲，1.6s 後塌縮爆炸。聚怪引爆流。`castBlackHole` + `updateBlackHoles`。
- **雷掌**：近戰電掌，命中擊退；**站在水裡放 → 整片水池導電**（自己也在水裡＝自傷搞笑）。成本：中。
- **冰錐突刺**：向前短距突刺 + 冰錐，命中緩速、可凍住前方水池。和冰衝刺同流。成本：中。
- **黑洞**：吸敵人/投射物/毒雲成一團再塌縮。聚怪引爆，超高分享感。成本：中～高。
- **冰牆 / 土牆**：見第一批（已排入）。
- （持續補充…）

> **近戰軸（值得開的一條線）**：目前主攻全是投射物；副攻池放近戰類（土拳頭 / 雷掌 / 冰錐突刺）+ 元素衝刺，就能開「貼身突進」流派，手感和「站遠放法術」完全不同 → 重玩性拉開。

> - **冰衝刺 + 土拳頭**：冰衝留滑面 → 滑進去一拳，把怪打進牆。

## 6. 設計守則（每次加內容都要過）

1. **不加第五個動作鍵**（手機）。
2. 新法術要能**碰到元素/地形或既有災難**（否則違背本作差異化）。
3. 每個法術配 **1～2 個升級池修飾符**，才進得了 roguelike build 循環。
4. 優先做**能兜出複合 combo**的東西（§2）。
5. **每批有界 + 接 ship gate**（見 roadmap.md），別讓內容拖延上架。

## 7. 架構彈性原則（夠維護就好，不過度設計）

> 目標：加一個法術/元素 = **多一筆資料 + 一個小函式**，不是動全身的手術。本作已是表驅動，沿用即可。

- **副攻插槽 = 註冊表**：每個副攻法術一筆 `{ id, name, cast(state, dir), cooldown, upgrades }`；按鍵只 dispatch 到「本局裝備的那一筆」。**禁止**在各處散落 `if (secondary === 'icewall')` 分支。
- **元素衝刺 = 一張表**：`element → { trail, onCross(tile) }`，加一個元素的衝刺行為＝加一筆，不改衝刺主流程。
- **元素/融合**：新元素（如風）只加進 `ELEMENT_INFO` / `fusionKind` / 顯示名表 + 對應反應處理；不要散寫條件。
- **地形反應**：牆＝寫進 `game.map` 的臨時 tile，重用既有碰撞/反應；新 tile 類型＝加常數 + 畫法 + 反應，集中在已知的幾處。
- **升級**：續用 `upgradePool` 陣列（`{ id, apply() }`），新修飾符＝一筆。
- **不要做的事**：別為了「未來擴充」就上 ECS / 外掛框架 / 通用事件總線。**抽象等到第三個同類出現再做（rule of three）**，在那之前「表 + 小函式」就夠。
- **維護提醒**：`index.html` 與 `camera-sandbox.html` 的共用邏輯要同步（後者是前者 + 攝影機面板）。
