# v2 規格 D — 競技場地圖 + 「搶獎盃 → Boss 甦醒 → 追逐」loop

> 對應：v2《環境處決 PvP》北極星「環境/Boss 當劊子手」；站在規格 A（蠢死法）的墜落演出上。
> 目的：給 v2 一個**有目標的局**(不只是互推) —— 玩家衝去搶獎盃,拿到瞬間 Boss 甦醒開始追,
> 全場邊逃邊用環境動詞把彼此送進 Boss / 坑裡。地圖本身就是劇本。
> 狀態：可執行規格。第一張落地目標 = **斷橋孤島**。

---

## 0. 一句話
**搶在最遠端的獎盃 → 拿到的瞬間沉睡 Boss 甦醒並追逐 → 地圖的危險(空洞/水/油/冰)是真正的殺手。** 拿到獎盃撐到時間結束 / 活到最後 = 贏。

---

## 1. 引擎現實:平面假裝 3D(務必先懂)

sim 跑在**單一平面**(960×640，30×20 格)。沒有真高低差。所以:

- **高低/浮島感 = 純美術假裝**;gameplay 永遠同一層。
- **做得到**:單一浮島 + 四周墜落(被轟出邊緣就掉下去);多座島用同手法重複擺。
- **做不到**(先不碰):站上不同高度的台地、真斜坡、跳躍。要做是大改 sim。

### 1.1 「浮島 + 周圍會掉下去」如何實現(render-only)

墜落**玩法**已存在(`overVoid → updateDeathTheater`:凸眼→縮小旋轉沉沒，見規格 A)。缺的只是**視覺**。改動全在 `render.js`:

1. **平台板塊化**:只把「有地板的格子」擠成一塊**有厚度的島**(頂面=磚貼圖,側面=岩壁/碼頭壁),而非鋪滿整個畫面的平面。
2. **島外不畫地板**:把現在的全屏暗背景換成**天空漸層 + 遠方水面/雲 + 霧**;島緣外即開闊下墜空間。
3. **邊緣=懸崖**:外圈**不再是實心 `TILE_WALL`**,改成地板直接接 `TILE_VOID`(深淵) → 被轟出邊緣 = 沿用現成沉沒動畫,只是掉進天空/海而非黑方塊。

順帶解決:舊的「遠牆上方黑帶」變成刻意的天空背景。
代價:render 中等改動,自包在 render.js,不動 sim。

> **碰撞模型差異**:傳統地圖外圈是 `TILE_WALL`(撞不出去)。浮島地圖**取消外牆**,地板邊緣直接鄰 `VOID`;正常走不會自己掉(玩家自保),但**擊退向量**能把人送出邊緣 → 墜落。走路自己走出去也會掉(浮島本來就該如此,且很好笑)。

## 2. 共用骨架(每張地圖都套)

| 元素 | 說明 |
|---|---|
| `◎ 出生點` | 近端(靠鏡頭側)。2–4 人各一。 |
| 中段危險區 | 空洞/水/油/草/冰 —— 逼玩家穿過才能拿獎盃。 |
| `▣ 獎盃台` | 最遠端焦點(鏡頭深處)。可見、誘人、難拿。 |
| `☠ 沉睡 Boss` | 壓在獎盃台下/旁。**拾取獎盃 = 甦醒**。 |
| 甦醒後 | Boss 開始**追逐**(複用衝鋒兵直撲 AI);全場進入逃殺。 |

**勝負(MVP 先最簡)**:拿到獎盃 + 活到計時結束 = 贏;被送死則出局/重生扣分。細節待玩法跑通再定。

**🥔 燙手山芋(本期先不做,標記保留)**:持獎盃者被 Boss 鎖定 + 減速 → 想丟給別人。這是 loop 的張力靈魂,等最簡版跑通再加(見 §6)。

## 3. 第一張:斷橋孤島 Broken Isles（落地目標）

最貼參考圖的浮島群 + 把「轟進洞」喜劇最大化,直接服務規格 A 的笑出聲驗證。

```
 圖例: ◎出生  ·地板  ✕深淵(墜落)  ‡薄牆橋(可破)  ▣獎盃  ☠沉睡Boss
        (鏡頭在下方往上看;遠端=畫面上方=獎盃所在)

        ✕ ✕ ✕ [遠島·▣☠] ✕ ✕ ✕
        ✕ ✕  ‡           ‡  ✕ ✕
   [左島·····]✕✕[中島······]✕✕[右島·····]
        ✕ ✕  ‡           ‡  ✕ ✕
   ◎ [近左島·····]  ✕✕  [近右島·····] ◎
              ‡                ‡
            (近端島接出生點)
```

- **3–5 塊地板島**被 `✕深淵` 切開,靠 `‡薄牆橋`(可破!)相連。
- 搶獎盃要過脆橋 → Boss 醒於遠島 → 邊逃邊把對手轟下橋、或**炸斷別人腳下的橋**。
- 無外牆,島緣即深淵。
- **主打**:墜落 + 斷橋(蠢死法最大化)。

**做這張需要的新東西**:浮島 render(§1.1)+ 獎盃/Boss loop(§5)。沙盒可先用 `training` 挖好 island 佈局驗 render,再進 v2。

## 4. 地圖型錄（後續,先存著）

每張都是**固定有界競技場**、正面望遠縮景鏡頭(`fov35/angle25`)。

**B. 油碼頭 Oil Dock** — 一大塊平台四周圍水,撒滿油漬+草+爆箱。Boss 一追,開闊地板變地雷區(點油=連環爆,火燒草蔓延)。主打:油/火/連環爆。最接近現在的平台,改動最小。

**C. 冰湖 Frozen Lake** — 大片冰(滑→慣性把人推超遠)+ 水道(導電=範圍電擊)。全場打滑收不住腳,一掌滑射進牆/洞。主打:冰滑慣性 + 水導電。需調冰滑手感。

**D. 環形高台 Ring Plateau** — 中央視覺墊高的高台放獎盃,外圈一圈深淵護城河,中間地板環。Boss 繞環追,大家邊繞圈邊把彼此擠向外圈。主打:走位 + 外圈處決。

**E.(進階)崩塌神殿 Collapsing Temple** — Boss 醒後地板**逐格塌成深淵**,空間越縮越小逼出決戰。需要小的「磚塊衰變」系統。排到玩法跑通後。

## 5. 需要的新零件（成本一覽）

| 零件 | 做法 / 捐贈來源 |
|---|---|
| **獎盃實體** | 新的小 entity(位置固定在獎盃台);render 一個發光體素。 |
| **拾取觸發** | 玩家碰到獎盃半徑內 → `game.trophyHolder = pid`、`game.bossAwake = true`。 |
| **Boss 甦醒** | 複用 `spawnBoss` / Boss AI;改成「拾取才生成/啟動」。 |
| **Boss 追逐 AI** | 複用 `updateCharger`(衝鋒兵直撲玩家)當底 → 持續鎖定最近(或持獎盃者,見 §6)。 |
| **浮島 render** | §1.1:板塊 + 天空背景 + 邊緣懸崖。最大的一塊,但純 render。 |
| **勝負/計時 HUD** | v2.js 自家迷你 HUD 擴充(已有比分,加倒數 + 獎盃歸屬)。 |

> 全部都站在現有捐贈庫上:墜落演出(A)、charger 追逐、boss、體素 render、v2 迷你 loop。

## 7. 落地順序 + DoD

1. **浮島 render**(§1.1):先在 `training` 沙盒把一塊島 + 天空背景 + 邊緣墜落做出來。
   - DoD:把假人轟出島緣 → 凸眼 → 掉進天空,讀起來是「浮島掉下去」而非「沉進黑方塊」。
2. **斷橋孤島佈局**(§3):多島 + 可破橋,擺進 v2。
   - DoD:能跨橋移動、橋可被炸斷、橋斷後過不去。
3. **獎盃 + Boss loop**(§5,最簡版,**不含燙手山芋**):
   - DoD:拿獎盃 → Boss 甦醒並追 → 被 Boss/環境送死 → 計時結束判勝負。
4. 找 3–4 人實玩,回到規格 A 的笑出聲門檻 + 看「搶獎盃→被追」是否製造新的混亂笑點。
5.(過了再考慮)燙手山芋 + 型錄其他圖。
