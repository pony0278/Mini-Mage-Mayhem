# 打磨階段計畫（C-lite 之後）— 手感 / 付費點 / 模型 / BR 地基

> 用途：把「上架前的遊玩打磨、打擊感、付費點、自訂模型、BR 地基」拆成**一階段一階段**的任務,每階段獨立驗收 + 提交再進下一個。
> 配套：上架缺口與 4 決策見 [shipping-readiness.md](shipping-readiness.md);路線取捨見 [roadmap.md](roadmap.md);畢業大絕見 [capstones.md](capstones.md)。
> 狀態：**C-lite 程式面已完成(W1 接縫 / W2 觸控 / W3 音效 / W4 英文 / W5 SDK)**;本文件接續規劃。

---

## 0. 原則（每階段都套用）

1. **一階段一個交付**:做完 → headless 驗證 → commit/push → 才進下一階段。
2. **每項標 BR 相關性**,決定要不要認真投資:
   - 🔵 **核心/共用**:在 `sim.js`,BR 伺服器直接重用 → 認真做。
   - 🟢 **客戶端可帶走**:同引擎演化成 BR 客戶端時多半能帶。
   - 🔴 **BR 丟棄品**:只服務單機/平台(美術、平台 SDK、付費 UI)→ 做到夠用就停,別鍍金。
3. **價值排序**:留存(手感)> 收入(付費點)> 第一印象(美術)。BR 丟棄品再炫也不移動 BR 進度。
4. **相依性先講清楚**:模型未到 → 卡住的階段就先不開。

## 2. 各階段任務

### P1 · 打擊感 pass 🔵（最先做:最便宜、留存 ROI 最高）
**目標**:每一拳/每一發都有「肉感」。
- [x] **hitstop / 頓幀系統 ✅**:`game.hitstop` 計時器,`update` 在 `playing` 守衛後 >0 就跳過該幀 gameplay 推進(cosmetics 照跑);`addHitstop(s)` 用 Math.max + 0.12s 上限防疊加卡頓。命中點:近戰連擊(土拳 0.06 / 雷掌 0.045 / 風掌 0.05 slam beat)、飛彈命中 0.03(Boss 0.05)、爆炸依半徑 0.045–0.1、擊殺 0.03(Boss 死 0.12)。
- [x] **🐛 修正:土拳一直崩潰** —— 模組拆分(4df80e8)後 `meleeAttack` 殘留呼叫 render 的 `colorHex`(sim 看不到)→ 土拳每拳丟 `colorHex is not defined` 卡死迴圈。改用 `parseInt(elColor.slice(1),16)` 在 sim 內轉色。**土拳現在真的能打了。**
- [x] **普攻自動突進 ✅**(P1.2)—— 左鍵近戰會鎖定**瞄準錐形內最近敵人**(150px、±~46°)、貼上去再揮(`nearestLungeTarget` + `p.atkLungeTime` 輕量突進,無 i-frame/不扣衝刺充能,撞牆受 collision 阻擋,命中後追擊被擊退的敵人)。解決「動作沒表現/打空」。沙盒木樁也修成會吃擊退。
- [x] **逐攻擊命中回饋微調 ✅**(P1.3):每次命中在接觸點噴**元素色火花叢 + 快速接觸環**(`hitSpark(x,y,color,power)`,全程序化、無模型);命中閃光改成**依傷害縮放**(`e.hurt = clamp(0.12 + dmg*0.0045, .., 0.3)`)→ 重擊比刮傷更亮更久。接線點:飛彈命中(`applySpellHit`,色用 coreColor、power 隨傷害)、土拳(power 1.8 最重)、雷掌(1.3)、風掌貼身(1.0)。headless 驗:土拳 16 傷→閃光 0.159、火球 36 傷→閃光 0.24,火花皆噴出,無錯誤。
- [x] **風掌完整重量 ✅**(P1.4):把「一圈圈」換成**方向陣風 + 硬反應**。① 視覺:移除圓環/放射粒子,改噴**沿瞄準方向、橫跨錐口扇形**的陣風條紋(22 粒順著 aim 流出),配合既有 slam 前向錐 → 一眼看出「風往那邊吹」。② 硬反應撞牆:被吹向牆的敵人在牆前吃**撞牆傷害 + 0.6s 暈眩 + 灰塵**(風的重量感來源)。③ 硬反應吹進火場:被推進火焰區的敵人**追加火傷 + 「燒!」**,讓「把敵人吹進火」成真連段。雲/箱推動改成**距離比例**。headless 驗(觸控 seam 鎖定瞄準、注入牆格):撞牆 ✓(暈 0.6)、燒 ✓、陣風 38 粒(22 順向)、敵 300→256、無錯誤。
- [x] **命中回饋微調 ✅**(併入 P1.1/P1.3):命中閃光(傷害縮放)、接觸粒子(`hitSpark`)、擊退(各近戰/`applySpellHit` 既有)、擊殺粒子噴散(`killEnemy` 既有 12 粒 + 菁英 18 粒 + 冰爆);大招/Boss 命中已加重 hitstop(Boss 命中 0.05 / Boss 死 0.12 / 大爆依半徑)。
- [x] **螢幕震動節流 ✅**:過去每個揮擊/施法/命中都直接 `Math.max` 進 `screenShake`,持戰時畫面從不沉澱=持續抖。改成 `addShake(s)`:**小震(< 6)**走冷卻節流(每 0.1s 只進一次)+ 軟上限 3.5;**大震(≥ 6,Boss/大爆/地震/絕對零度…)**繞過兩者、照常重擊。38 個呼叫點全改走 `addShake`(reset/decay 不動)。headless 驗:持續土拳 max 3(≤3.5 上限)、放手後沉澱 0;毒爆 8.32 穿透節流;無錯誤。
- [x] **🐛 修正:粒子從來沒被畫出來 + 環永遠是白色 ✅**(調查近戰視覺時發現):①`game.particles` 全程被模擬/更新,但 3D renderer **沒有任何 draw pass**(連模組化前的 index.html 也沒有)→ 火花、粉塵、碎石、P1.4 陣風、死亡噴散**全部隱形**。在 `syncZones` 補上粒子渲染(小體素方塊、`colorHex(色)`、非疊加混色避免堆疊爆白)。②`game.rings` render 寫死 `0xffffff` → 每個環都是白圈;改用 `colorHex(rg.color)`。③ slam 的白色接觸閃光/內環改吃 stance 色(只留小白熱點),土拳預設色給土黃 `0xc89a5a`。**附帶效果:P1.3 火花、P1.4 陣風現在才真的看得到。**
- [x] **土拳土系意象 ✅**:命中改噴**深棕碎石塊(8 粒、大顆、非疊加)+ 揚塵 + 土黃衝擊環/錐**,取代原本的白圈。截圖驗證:白圈消失,讀作泥土重拳。
- [ ] **(選)大命中微慢動作**:Boss 致命一擊/大爆短暫放慢。
**驗收**:hitstop 在近戰/飛彈/爆炸都觸發;無效能回退;headless 驗計時器邏輯。

### B · 近戰招牌差異化 🔵（第二層:依 melee-combo-reference 的「招牌差異化」路線,CP 值最高）
**取捨**:不為三流派各建完整「3 段普攻 + 重擊」(分段預備/命中/收招動畫要等 **P3 模型**才到位,現在硬做體素動畫投報率低)。改為**先給每個流派一個不可替代的招牌機制**(文件核心準則:土改地形 / 雷改狀態 / 風改位置)。全在 `sim.js` → 🔵 BR 重用,不是丟棄品。
- [x] **土系招牌(改地形)✅**:撞牆追加傷害 + breakThinWalls + 砸箱碎片 + ★2 震波 + ★3 地震(P1 前既有)。
- [x] **風系招牌(改位置)✅**(P1.4):方向陣風 + 撞牆暈眩 + 吹進火場追傷 + 推箱/雲。
- [x] **雷系招牌(改狀態)✅**:**雷印 + 雷步穿身**。雷掌/雷鏈替敵人掛「雷印」(`e.lightMark`,4s,render 青色閃爍環);雷掌模式(或帶雷元素的衝刺)**穿過**帶印敵人 → 消耗雷印引爆**雷爆 + chainLightningFrom 跳電**(獨立寬判定 `p.zapHits`,即使突進停短也能引爆;**無地面電區=不自傷**,沿用 venomnet 教訓)。複用 dash 鍵,無新按鍵/動畫。卡片敘述 + i18n(雷爆!/雷步穿身/風助火勢/撞牆!/燒!)同步。headless 驗:雷掌命中→lightMark 4、衝刺穿過→「雷爆」+ 扣血,無錯誤。

- [ ] **(之後)連斬刷新**:殺掉帶雷印敵人 → 刷新衝刺充能(需設限防無限)。
- [ ] **(之後/需模型)3 段普攻分段**:右勾拳/橫斬/旋掌的分段節奏;分段動畫等 P3。
- [ ] **(之後)跨系互動深化**:帶電箱、碎石雷爆、風吹毒火(部分既有)。
