# 打磨階段計畫（C-lite 之後）— 手感 / 付費點 / 模型 / BR 地基

> 用途：把「上架前的遊玩打磨、打擊感、付費點、自訂模型、BR 地基」拆成**一階段一階段**的任務,每階段獨立驗收 + 提交再進下一個。
> 配套：上架缺口與 4 決策見 [shipping-readiness.md](shipping-readiness.md);路線取捨見 [roadmap.md](roadmap.md);畢業大絕見 [capstones.md](capstones.md)。
> 狀態：**C-lite 程式面已完成(W1 接縫 / W2 觸控 / W3 音效 / W4 英文 / W5 SDK)**;本文件接續規劃。

---

## 0. 原則（每階段都套用）

1. **一階段一個交付**:做完 → headless 驗證 → commit/push → 才進下一階段。
2. **每項標 BR 相關性**,決定要不要認真投資:
   - 🔵 **核心/共用**:在 `sim.js`,BR 伺服器直接重用 → 認真做。
   - 🟢 **客戶端可帶走**:同引擎演化成 BR 客戶端時多半能帶。
   - 🔴 **BR 丟棄品**:只服務單機/平台(美術、平台 SDK、付費 UI)→ 做到夠用就停,別鍍金。
3. **價值排序**:留存(手感)> 收入(付費點)> 第一印象(美術)。BR 丟棄品再炫也不移動 BR 進度。
4. **相依性先講清楚**:模型未到 → 卡住的階段就先不開。

## 2. 各階段任務

### P1 · 打擊感 pass 🔵（最先做:最便宜、留存 ROI 最高）
**目標**:每一拳/每一發都有「肉感」。
- [ ] **hitstop / 頓幀系統**:命中(近戰/飛彈/爆炸)瞬間凍結 sim 幾十毫秒,依傷害量縮放;在 `update` 之前讀一個 `game.hitstop` 計時器,>0 時跳過該幀推進(或放慢)。
- [ ] **命中回饋微調**:命中閃光 / 接觸粒子 / 擊退曲線 / 擊殺「碎裂」感;大招/Boss 命中加重 hitstop。
- [ ] **螢幕震動節流**:避免小事狂震、大事才重震。
- [ ] **(選)大命中微慢動作**:Boss 致命一擊/大爆短暫放慢。
**驗收**:hitstop 在近戰/飛彈/爆炸都觸發;無效能回退;headless 驗計時器邏輯。
**BR**:🔵 全在 sim,BR 重用。

### P2 · 獎勵廣告付費點 🔴（便宜、直接帶收入/互動數據）
**目標**:用 rewarded 廣告做玩家自願的付費點(run-based、不需 meta 貨幣)。
- [ ] `platform.js` 加 `rewardedAd(onReward)`:只有 `adFinished` 才發獎;無 SDK 時回報「不可用」讓 UI 隱藏按鈕。
- [ ] **復活**(死亡畫面):看廣告 → 半血續命、清掉身邊威脅,一局一次。
- [ ] **重抽升級**(升級畫面):看廣告 → 換一組新的 3 張卡(或選 2 張)。
- [ ] **(選)免費換武器**:看廣告 → 免費多注入一個元素/副攻/肉搏(你的「換武器」點子)。
- [ ] UI:over/upgrade 畫面的廣告按鈕(觸控+點擊),依 SDK 可用性顯示/隱藏。
**驗收**:stub SDK 下獎勵正確發放;無 SDK 優雅隱藏;復活/重抽正確改 sim 狀態。
**BR**:🔴 付費 UI 是單機/平台(BR 走 Nakama meta);但 hook 便宜、帶 C 的收入數據。

### P3 · 自訂模型 + 攻擊動畫 🔴（卡相依:需先放模型檔)
**前置(你)**:把模型放進 `assets/models/`(或給路徑),最好 **GLB**、低面數、含 idle/run/**attack**(+hurt/death) clip。
- [ ] vendor **GLTFLoader**(核心 three r144 沒有)+ `AnimationMixer`。
- [ ] 載入管線:載 GLB → 快取 → 把 sim 狀態(待機/移動/攻擊/受傷/死亡)對應到動畫 clip。
- [ ] 先換**玩家**體素 mesh → 再敵人/Boss(若有模型)。
- [ ] 確認固定 45° 俯視下的辨識度、載入體積、手機效能。
**驗收**:模型載入、依狀態播動畫、檔案/效能可接受。
**BR**:🔴 純美術,BR 丟棄 → 做**適量**、別鍍金。**與 P4 綁**(肢體破壞需分件模型)。

### P4 · Boss 肢體破壞 + 戰鬥深化 🔵/🔴
**目標**:Boss 戰更有深度與爽度。
- [ ] **簡版(現在就能做)**:Boss 受擊敲掉體素塊、階段性拆解、部位視覺改變。
- [ ] **完整版(需 P3 分件模型)**:各部位獨立 HP,打掉一個部位 → 停用對應招式 + 視覺改變。
- [ ] 連動 P1 的 hitstop/回饋,讓 Boss 命中更重。
**驗收**:部位可破壞、招式對應停用、視覺回饋清楚。
**BR**:邏輯(部位 HP/招式)🔵 重用;模型分件 🔴。

### P5 · B0 收尾（種子化 RNG + 純 step）🔵（BR 真正起跑線,可並行)
**目標**:讓 `sim` 完全可決定性、可在伺服器跑。
- [ ] 種子化 PRNG 取代 `Math.random()`,串進 sim(`rnd` 等)。
- [ ] `update` 收成純 `step(state, inputs, dt)`(無 DOM、無模組級可變狀態外溢)。
- [ ] 決定性測試:同 seed + 同 inputs → 逐幀一致。
**驗收**:決定性測試通過。之後即可接 [battle-royale-phase-plan-v1.md](battle-royale-phase-plan-v1.md) 探針一/二。
**BR**:🔵 這就是地基。**與美術無關,任何時候可並行。**

### P6 · 上架 CrazyGames + 收留存數據 🔴（你的手動步驟)
- [ ] 對 CrazyGames QA 要求微調(SDK 載入、廣告政策、banner/rewarded)。
- [ ] 上傳、過審、上線。
- [ ] 收**留存/回鍋/分享**數據 → 餵 roadmap §4 的 BR Go/No-Go。
**建議**:P1(手感)+ P2(付費點)做完再上,數據才反映「調過且有變現的版本」。
