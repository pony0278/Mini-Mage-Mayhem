# 俯視角魔法 Roguelike 小品企劃

> 暫定方向：**Wizard of Legend + Spell Disk + Magicraft + Noita 的簡化版**  
> 目標平台：Poki / CrazyGames / Web Demo  
> 核心體驗：**3～5 分鐘一局，用隨機法術組合與元素反應，把地牢打成一團混亂並活下來。**

---

## 1. 核心定位

這是一款 **短局制俯視角魔法動作 Roguelike 小品**。

玩家扮演一名小法師，被丟進小型地牢或競技場中。每局透過隨機獲得的法術與修飾符組合，打敗怪物、觸發元素連鎖反應、改變戰場，最後擊敗小 Boss 或撐過一定時間。

遊戲不是大型 RPG，也不是完整 Noita，而是：

> **Wizard of Legend 的俯視角動作手感**  
> + **Spell Disk / Magicraft 的法術 Build 爽感**  
> + **Noita 的元素混亂與場地反應**  
> 壓縮成一局 3～5 分鐘的網頁小品。

---

## 2. 參考遊戲拆解

| 參考遊戲 | 取用重點 | 不照搬的部分 |
|---|---|---|
| **Wizard of Legend** | 俯視角戰鬥、閃避、短距離施法、房間戰鬥節奏 | 不做太多精細連招與高操作門檻 |
| **Spell Disk** | 法術之間互相觸發、Build 連鎖、每局不同組合 | 不做過於複雜的自動化觸發系統 |
| **Magicraft** | 法術 + 修飾符組合，逐漸失控的爽感 | 不做大型背包式法術編輯器 |
| **Noita** | 元素反應、火水毒雷爆炸、場地破壞與意外死亡 | 不做每個像素的完整物理模擬 |

---

## 3. 一句話玩法

> **在 5 分鐘內，用隨機法術組合，把整個地牢炸成一團混亂，活下來。**

---

## 4. 單局流程

### 一局時間

- 目標：3～5 分鐘
- 適合網頁平台短局體驗
- 死亡後快速重開

### 流程設計

1. **開局選一個基礎法術**
   - 火球
   - 冰箭
   - 雷鏈
   - 毒彈
   - 風刃
   - 石彈

2. **進入小地圖 / 小地牢**
   - 3～5 個小房間
   - 或一張封閉競技場
   - 敵人分波出現

3. **每打完一波，選一個升級**
   - 法術分裂
   - 法術變大
   - 命中爆炸
   - 留下火焰 / 毒霧 / 冰面
   - 冷卻縮短
   - 擊殺後觸發額外效果

4. **中後段開始失控**
   - 火燒草地
   - 雷電導通水池
   - 毒霧遇火爆燃
   - 冰凍水面造成滑倒
   - 爆炸破壞薄牆或木箱

5. **最後階段**
   - 打小 Boss
   - 或撐過最後一波敵人
   - 勝利後結算金幣 / 解鎖新法術
   - 死亡後給少量資源並快速重開

| 左鍵 | 主法術 |

手機版可以之後再做，不建議第一版就完整支援。

| 操作 | 功能 |
|---|---|

| 技能按鈕 | 副法術 / 特殊法術 |

## 6. 核心玩法支柱

### 6.1 動作戰鬥

玩家不是站樁放技能，而是需要：

- 走位
- 閃避
- 引怪
- 利用場地
- 控制元素反應
- 避免被自己的法術害死

### 6.2 法術 Build

每局會逐漸獲得新修飾符，讓基礎法術變得更誇張。

例如：

- 普通火球 → 大火球
- 大火球 → 爆炸火球
- 爆炸火球 → 分裂火球
- 分裂火球 → 留下火焰路徑
- 擊殺敵人後再次爆炸

玩家會感覺自己的 Build 逐漸失控，形成爽感。

### 6.3 元素反應

這是本遊戲最重要的差異化設計。

不是只有「技能打怪」，而是玩家可以透過魔法改變戰場。

例如：

- 火燒草地
- 雷電導通水池
- 毒霧遇火爆炸
- 冰凍水池形成冰面
- 風把火焰或毒霧吹開
- 爆炸打破薄牆

### 6.4 搞笑災難

遊戲應該允許玩家因為自己的法術死掉。

例如：

- 打爆油桶，燒死敵人，也燒到自己
- 雷電打進水池，敵人全死，玩家也被電到
- 毒霧被火點燃，整個房間爆炸
- 爆炸打開薄牆，結果釋放更多敵人

這會創造出影片感與分享感。

## 7. 元素反應系統

第一版不做完整 Noita 的像素物理，而是做簡化版格子 / 區域反應。

| 元素互動 | 效果 |
|---|---|
| 火 + 草地 | 草地燃燒，形成火區 |
| 火 + 油 | 大範圍爆燃 |
| 火 + 毒霧 | 毒霧爆炸 |
| 水 + 雷 | 水區導電，造成範圍傷害 |
| 水 + 冰 | 變成冰面，敵人滑倒或減速 |
| 風 + 火 | 火焰向指定方向擴散 |
| 風 + 毒霧 | 毒霧被吹散 / 推開 |
| 爆炸 + 薄牆 | 牆壁被破壞 |
| 石頭 + 火 | 形成熔岩碎片 / 火焰碎片 |

### MVP 優先實作反應

第一版只需要 4 種：

1. **火燒草地**
2. **火引爆毒霧**
3. **雷電導通水池**
4. **爆炸破壞薄牆**

這 4 個就足以展示核心魅力。

## 8. 地圖設計

### 地圖規模

第一版不需要大型隨機地牢。

建議：

- 20 × 20 或 30 × 30 的俯視角小地圖
- 手工設計 1～3 張地圖
- 每局隨機抽一張

### 地圖元素

| 元素 | 功能 |
|---|---|
| 石牆 | 阻擋移動與投射物 |
| 薄牆 | 可被爆炸破壞 |
| 木箱 | 可破壞，可能掉落資源 |
| 草地 | 可被火點燃 |
| 水池 | 可導電、可被冰凍 |
| 油桶 | 被火擊中後爆燃 |
| 毒桶 | 被破壞後產生毒霧 |
| 敵人生成點 | 分波出怪 |
| Boss 區域 | 最後階段戰鬥 |

## 9. 法術系統

第一版不要做太多法術，先做少量但能互動的技能。

### 9.1 基礎法術

| 法術 | 特色 |
|---|---|
| 火球 | 爆炸、燃燒、點燃場地 |
| 冰箭 | 緩速、冰凍水面 |
| 雷鏈 | 導電、連鎖攻擊 |
| 毒彈 | 產生毒霧、持續傷害 |
| 風刃 | 擊退、吹散火焰與毒霧 |
| 石彈 | 高傷害、破牆、擊退 |

### 9.2 MVP 法術

第一版建議只做 3 個：

1. **火球**
2. **雷鏈**
3. **毒彈**

這三個可以直接支撐元素反應：

- 火 + 毒霧 = 爆炸
- 雷 + 水池 = 導電
- 火 + 草地 = 燃燒

### 9.3 修飾符

| 修飾符 | 效果 |
|---|---|
| 分裂 | 法術分裂成多發 |
| 反彈 | 碰牆反彈 |
| 穿透 | 穿過敵人 |
| 爆炸 | 命中後爆炸 |
| 追蹤 | 微弱追蹤敵人 |
| 擴散 | 變成扇形攻擊 |
| 留痕 | 飛行路徑留下元素 |
| 擊殺觸發 | 擊殺敵人後再次觸發法術效果 |
| 冷卻縮短 | 施法頻率提高 |
| 範圍增加 | AoE 變大 |
| 推力增加 | 擊退更強 |
| 自傷加倍效果 | 法術更強，但也會傷到自己 |

### MVP 修飾符

第一版建議只做 6 個：

1. 分裂
2. 爆炸
3. 留痕
4. 冷卻縮短
5. 範圍增加
6. 擊殺觸發

## 10. Build 範例

### 火球 Build

初始：

> 發射一顆火球，命中敵人造成傷害。

升級後：

- 火球變大
- 火球分裂成 3 顆
- 火球命中後爆炸
- 火球飛行路徑留下火焰
- 擊殺敵人後再次爆炸

結果：

> 玩家變成移動火災製造機，整個房間開始燃燒。

---

### 雷鏈 Build

初始：

> 發射一道雷電，命中敵人後連鎖到附近敵人。

升級後：

- 雷電連鎖次數增加
- 水池導電範圍變大
- 擊殺後再次觸發雷鏈
- 冷卻縮短

結果：

> 玩家可以利用水池清場，但自己站在水中也可能被電到。

---

### 毒彈 Build

初始：

> 發射毒彈，命中後產生毒霧。

升級後：

- 毒霧範圍增加
- 毒彈分裂
- 毒霧被火點燃後爆炸
- 擊殺敵人後留下毒霧

結果：

> 房間逐漸被毒霧覆蓋，玩家需要用火引爆或用風清場。

## 11. 敵人設計

第一版敵人不需要太多，重點是逼迫玩家移動與利用場地。

| 敵人 | 行為 | 設計目的 |
|---|---|---|
| 史萊姆 | 慢慢靠近玩家 | 基礎近戰壓力 |
| 火焰小鬼 | 遠距離丟火球 | 逼玩家閃避 |
| 毒蟲 | 死後留下毒霧 | 製造場地危險與元素反應 |
| 石頭怪 | 血厚、衝撞 | 打亂走位 |
| 雷法師 | 遠距離放雷 | 配合水池製造危險 |

### MVP 敵人

第一版只做 3 種：

1. 史萊姆
2. 火焰小鬼
3. 毒蟲

## 12. Boss 設計

### 元素哥布林法師

第一版小 Boss 可以是一隻會展示元素反應的哥布林法師。

#### 技能

- 召喚火圈
- 丟毒瓶，產生毒霧
- 召喚水池
- 對水池放雷
- 召喚小怪
- 半血後場地隨機出現元素桶

#### 設計目的

這隻 Boss 不是單純高血量，而是讓玩家理解：

> 場地本身也是戰鬥的一部分。

玩家可以利用 Boss 召喚的水池、毒霧、火焰反過來打 Boss。

## 13. 遊戲爽點

### 13.1 自己製造災難

玩家不是只把敵人打死，而是會意外製造大混亂。

例如：

> 打爆油桶 → 敵人全燒死 → 玩家自己也差點被火燒死。

這種體驗很容易形成記憶點。

---

### 13.2 Build 越來越失控

一局的節奏應該是：

- 第 1 分鐘：普通火球
- 第 2 分鐘：火球會爆炸
- 第 3 分鐘：火球分裂 + 留火焰
- 第 4 分鐘：擊殺敵人後連鎖爆炸
- 第 5 分鐘：整個房間變成火海

玩家會覺得自己逐漸變強，也逐漸失控。

---

### 13.3 場地被魔法改變

玩家應該能明顯感覺到場地在變化：

- 草地被燒光
- 牆壁被炸開
- 水池變成導電區
- 毒霧擴散
- 冰面讓敵人滑倒

---

### 13.4 快速重開

死亡不應該讓玩家挫折太久。

死亡後：

- 顯示本局 Build
- 顯示擊殺數
- 顯示造成最大災難的元素反應
- 給少量金幣或解鎖進度
- 一鍵重開

---

## 14. 為什麼適合 Poki / CrazyGames

| 特性 | 適合原因 |
|---|---|
| 3～5 分鐘一局 | 符合網頁平台短局節奏 |
| 操作簡單 | WASD + 滑鼠 + 少量技能即可 |
| 法術組合 | 提供重玩性 |
| 元素反應 | 有記憶點與影片效果 |
| 單機可玩 | 不必一開始做伺服器 |
| 死亡重開快 | 適合 casual 玩家 |
| 可逐步擴充 | 後續能加法術、角色、地圖、Boss |

## 15. 跟原本「家園經營 + 掠奪 + Roguelike」方向比較

| 原本方向 | 問題 |
|---|---|
| 家園經營 | 系統多，初期展示爽感較慢 |
| 非同步 PvP 掠奪 | 需要伺服器、資料保存、防作弊、平衡 |
| Replay / 防守配置 | 開發成本較高 |
| 長線養成 | 內容量需求大 |

| 魔法 Roguelike 小品 | 優點 |
|---|---|
| 單機可玩 | 開發門檻低很多 |
| 玩法一看就懂 | Demo 展示效果好 |
| 法術與元素反應 | 容易做出記憶點 |
| 短局 | 適合網頁平台 |
| 可小規模驗證 | 不需要大量內容就能測手感 |

建議先用這個方向做 demo，驗證：

> **法術組合 + 元素反應 + 短局戰鬥是否真的好玩。**

## 19. 美術方向

- 俯視角 / 2D 或 2.5D
- 小法師 Q 版角色

| 元素 | 視覺要求 |

## 22. 第一版成功標準

如果 demo 達到以下條件，就值得繼續擴充：

- 玩家第一次玩 30 秒內懂操作
- 玩家 1 分鐘內看到元素反應
- 玩家 3 分鐘內 Build 開始失控
- 玩家死亡後願意馬上重玩
- 觀眾看影片能立刻理解「這遊戲會製造魔法災難」

## 23. 最終方向總結

這款遊戲最重要的不是魔法題材本身，而是：

> **玩家每一局都能組出一個失控、搞笑、很爽的法術災難。**

第一版不要做大，不要做多人，不要做完整物理。

只要先做出：

- 短局
- 俯視角戰鬥
- 3 個法術
- 6 個修飾符
- 4 個元素反應
- 1 張地圖
- 1 隻 Boss

就已經可以做出一個很有展示力的 Poki / CrazyGames demo。
