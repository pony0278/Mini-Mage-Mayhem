# 畢業組合（Capstones）— 設計記錄

> 用途：記錄「build 畢業大絕」的設計 —— 一套 build 投資到位後解鎖的強力 capstone（例：火+土 → 流星降臨）。
> 配套：emergent 自發組合的哲學見 [content-backlog.md](content-backlog.md) §2；玩法現況見 [spells-and-upgrades.md](spells-and-upgrades.md)；方向取捨見 [roadmap.md](roadmap.md)。
> 狀態：**設計討論完成、尚未實作**。這是「顯式組合 / 畢業卡」方向，**刻意補在 emergent 之上**給 build 一個爽的終點。

---

## 0. 一句話

> 一套 build 投資到位 → 解鎖一個**被動觸發的畢業大絕**，把這套 build 的幻想開到 11。
> capstone 是內容（每個都要設計/平衡/特效/預警），所以**分階上、一局只畢業一條**。

---

## 1. 框架（統一規則，全部重用現有系統）

- **解鎖**：就是一張**高階升級卡**，靠 `upgradeRelevant()` 門控 —— 只在「**擁有該組合 + 波數夠高（例 ≥3）+ 還沒拿過**」時才出現。完全沿用升級系統，零新機制。
- **觸發**：**被動 proc / 充能自動放**。守鐵則「**不加第五個動作鍵**（手機友善，見 content-backlog §0）」。觸發條件例：每隔 N 秒、每殺 N 隻、HP < X%。
- **公平性稅**：任何「砸地 / 範圍災難」型 capstone **一律先用落點預警**（重用 `addBossWarning` 的預警圈），否則玩家會覺得莫名暴斃（見 design-vision §9）。
- **節制 / 互斥**：**一局通常只畢業一條** —— 拿了一個 capstone 後，其他 capstone 的出現率下降或互斥，避免後期變煙火秀 + 公平性崩盤。
- **架構**：capstone = `upgradePool` 的一筆 `{ id, name, desc, apply, requires }`；`apply` 設一個 `stats.capstone.*` 旗標，sim 的某個 tick 讀旗標 proc。**新增一個 capstone = 加一筆資料 + 一段 proc**（沿用表驅動）。

---

## 2. 全菜單：10 融合 × 放大各自身份

每個大絕都是把該融合**既有身份**開到 11（畢業感），不是亂塞新機制。

| 融合 | 現有身份 | 畢業大絕 | 重用 |
|---|---|---|---|
| 土+火 熔岩 | 重擊+岩漿 | **流星降臨**：週期天降流星，落點爆炸+岩漿池 | 爆炸+火區+預警 |
| 火+毒 毒爆 | 延遲爆破 | **瘟疫核爆**：你佈的毒霧週期自動連環引爆 | 既有毒爆鏈 |
| 火+雷 電漿 | 高風險爆發 | **電漿風暴**：遊走電漿球，經過處爆裂+導電 | 爆炸+雷鏈 |
| 火+冰 蒸氣 | 遮蔽濕氣 | **沸騰領域**：大範圍蒸氣風暴，全場減速+灼燒 | 蒸氣雲+減速 |
| 雷+冰 電霜 | 麻痺控場 | **絕對零度**：週期冰凍新星，凍結+電擊全場 | 凍結+電區 |
| 雷+毒 電毒 | 持續麻痺 | **劇毒電網**：身邊持久帶電毒場，踩到麻痺+中毒 | 毒雲+電區 |
| 冰+毒 毒冰 | 陷阱地形 | **凍毒領域**：環身光環，範圍內持續凍+毒 | 凍+毒 |
| 土+冰 凍岩 | 減速+破牆 | **冰川崩落**：生成冰牆群 → 同時碎裂大範圍冰爆 | 冰牆+冰爆 |
| 土+雷 磁暴 | 聚怪+雷鏈 | **磁暴奇點**：週期磁力黑洞吸成一團 → 雷鏈引爆 | 黑洞+雷鏈 |
| 土+毒 毒沼 | 毒沼 | **大地崩毀**：腳下毒沼持續擴張 + 迸碎石 | 毒沼+碎石 |

> 單元素是否也有 capstone（靠高元素精通解鎖）= 待議；目前先以**融合對**為主。

## 3. 另外兩條軸：肉搏 / 副攻的「星級進化」畢業

這些走**重複投資 → ★**（同元素精通/副攻精通的慣用法）。風掌就是這條的起點。

| build | ★ 進化 → 畢業 |
|---|---|
| **風掌** | ★1 舉一個丟（現況）→ ★2 **多重撿取齊射** → ★3 **舉場上的牆丟** → 畢業「風暴投擲」 |
| **土拳** | ★ 重擊產生**震波** → 畢業「地震」全場震退+破牆 |
| **雷掌** | ★ 每次命中追加雷鏈 → 畢業「雷神」站著放電全場 |
| **黑洞（副攻）** | ★ 塌縮升級成「**超新星**」特大爆 |

> 升星機制：把「重選當前肉搏模式 → +1★」打開（目前 `upgradeRelevant` 擋掉重選當前模式；改成像副攻強化那樣 +★）。
> 風掌 ★3「舉牆」較難：牆是寫進 `game.map` 的 tile，要「把 tile 變成可丟的碎塊 prop」（keystone prop 系統撐得住，但屬中偏難）。

---

## 4. 必須認清的現實

1. **這是大內容**：全做 ≈ 14 個大絕，每個要設計 + 平衡 + 特效 + 預警。**不可能一次做完。**
2. **分階上**：先做 1 個 Tier-0 試金石（驗證 capstone 框架），再每批加 2–3 個。
3. **互斥 / 節制**：一局只畢業一條，否則後期變煙火秀、公平性崩。
4. **與 emergent 的關係**：capstone 是**顯式**獎勵，刻意疊在 emergent（content-backlog §2）之上 —— 不取代它。別把所有 combo 都做成 capstone，否則失去「自己發現」的樂趣。

---

## 5. Tier-0 建議（先做這 3 個驗證框架）

挑**身份鮮明 + 重用最高 + 互不重疊**的當第一批：

1. **流星降臨**（土+火）— 純重用（爆炸+火區+預警），最好做；使用者原始點子。
2. **瘟疫核爆**（火+毒）— 幾乎 100% 重用既有毒爆鏈，近乎免費，強化「佈場」核心身份。
3. **風掌 ★2（多重撿取齊射）**— 驗證「肉搏星級」那條軸（先不碰 ★3 舉牆的難點）。

跑通後，capstone 框架（門控 + proc + 預警 + 互斥）就定型，之後加大絕只是往表裡塞。

---

## 6. 待拍板（動工前）

- **解鎖門檻**：波數 ≥N？元素精通 ≥M？還是純「擁有該組合」即可？
- **互斥強度**：硬互斥（拿一個就鎖其他）還是軟（出現率下降）？
- **觸發節奏**：proc 的冷卻/條件怎麼定，才不會洗版又不會沒存在感。
- **Tier-0 範圍**：就上面 3 個，還是先只做 1 個（流星）跑通框架再說？
- **單元素 capstone** 要不要做（靠高精通解鎖），還是只做融合對。
