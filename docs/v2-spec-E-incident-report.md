# v2 規格 E — 魔法事故報告 / 收容測試（北極星再定位）

> 來源願景：[magic-incident-report-concept.md](magic-incident-report-concept.md)（完整概念）。
> 本檔＝**可執行 spec**：把概念釘成定位/角色/收容狀態機/報告欄位/版本路線，並對照現有 codebase 的可重用零件。
> 關係：**升級並吸收規格 C（分享）為核心**；規格 A（蠢死法）/B（ownership）/D（地圖）降為「餵報告的引擎與素材」。

---

## 0. 一句話再定位

不是「打怪/互推的魔法亂鬥」，而是 **「魔法實驗基地裡的收容亂鬥，每局產生一份荒謬的魔法事故報告」**。
護城河＝**輸了也好笑、好笑到想截圖分享**。報告是核心產物，戰鬥是產生報告的引擎。

> 北極星驗收：玩家玩完一局會說「**這局也太白癡了，我要給朋友看**」並真的去截圖/複製分享文字。

## 1. 與既有 v2 規格的關係（不打掉重練）

| 既有 | 在新定位下的角色 |
|---|---|
| 規格 C（災難回顧/分享卡）| **被吸收為核心** → 就是「事故報告」。 |
| 規格 A（蠢死法：墜落/凸眼）| 事故的**演出層**（墜落=一種事故結局）。 |
| 規格 B（鋪設→引爆 ownership）| 事故的**因果來源**（你鋪的油/電炸到自己＝事故）。 |
| 規格 D（斷橋孤島）| **一種場地**；新增「魔法實驗基地」室內場地與之並列。 |
| 元素 spec | 不變；元素＝事故來源（§4 事故表）。 |

**結論**：報告(C)從「增長引擎」升為「主軸」；A/B/D 變成餵報告的素材。

## 4. 收容狀態機 — V0.9 對稱原型（定案：對稱 / 推進 / 簡化削弱）

**對稱**：雙方都能削弱並收容對方，無固定「失控法師/收容員」身份（不對稱版＝後續 V0.9b）。

```
削弱(失衡) → 陣風把失衡對手推進實驗艙 → 關艙 3 秒倒數 → 收容成功(對方該回合輸)
                                       ↑ 掙脫 / 捕方被轟離 / 危險命中艙 → 過載爆炸(反轉) ↓
```

| 旋鈕 | 起始值 | 說明 |
|---|---|---|
| 穩定值 STABILITY | 100 | 被陣風命中 −45、被環境(爆/電)−30；每秒回復 +28 |
| 失衡門檻 | <30 | 進入「失衡」：減速 40% + 頭上冒星 + **可被推入艙** |
| 實驗艙 | 原獎盃台改造的艙區(中央島) | 失衡者被推入艙半徑 → 開始關艙 |
| 關艙倒數 LOCK_T | 3.0s | 被收容者連打掙扎累積掙脫值 ≥ ESCAPE(100，約 1.5s) → 掙脫 |
| 中斷/過載 | 倒數中「捕方被轟離艙旁」或「爆炸/電擊命中艙」 | → 過載爆炸：全場擊飛 + 該次收容失敗(招牌反轉) |
| 勝負 | 收容成功 → 對方該回合輸；先贏 WIN_TARGET(3) 回合 → 事故報告 | |
| Boss | 本原型**先移除** | 收容迴圈本身夠亂；之後可加回當中立危險 |

> 解鎖的招牌事故(餵報告)：**反向收容**(剛收容完馬上被收容)、**艙門短路**(倒數被打斷過載)、
> **自行入艙**(被擊退/滑進艙)、**過載爆炸**(關艙失敗炸飛全場)。
> 笑點來源：推人被反炸、冰面雙方一起滑走、掙脫後反推對方進艙、雷短路艙門。

### 4.1 V0.9 playtest 回饋（✅ 好玩門檻通過 — 自己玩就很好玩）

> 第一次實玩結論：**核心成立**(收容迴圈 + 事故報告自己玩就停不下來)。以下為**延後微調**清單(不急)：

## 5. 事故報告 schema（V0.8 先做；最高 CP）

一局結束 → 依行為產生報告。欄位：

```
事故編號 #NNN ・ 事故名稱 ・ 事故等級(E~S+)
受測體 / 收容員 ・ 地點 ・ 使用法術
事故摘要(模板句拼裝) ・ 勝負
基地損害% ・ 誤傷次數 ・ 自傷次數 ・ 最高連擊 ・ 逃脫/收容時間
收容成功/失敗原因 ・ 安全委員會評語(吐槽) ・ 玩家稱號
挑戰碼 / 分享文字
```

**等級＝荒謬度**(非單純勝負)：E 輕微 → D 小事故 → C 場地損壞 → B 危險收容 → A 嚴重 → S 災難級 → S+ 禁術級。
規則範例：自傷≥1→至少D；觸發元素連鎖→B；收容員被關→S；同一局≥5種事故→「安全規範不存在」。

**事故名稱/稱號**：查表觸發(見概念文 §13/§14)。先做 10 條(對應事故圖鑑 #001~#010)。

**評語**：吐槽模板庫，依結果挑句(「技術上來說，有人被成功收容了。只是收錯人。」)。

## 6. 場地：魔法實驗基地（與浮島並列的新場地）

房間化危險場(概念文 §8)：實驗艙室 / 元素反應室 / 冷卻水池 / 雷電導線區 / 魔法倉庫(爆桶) / 失控核心房 / 傳送門走廊。
**原則**：不是公平空曠，是「**可理解的混亂**」——玩家看得出「這裡危險、但我能利用、失控會很好笑」。
MVP 不必全做；先一個「實驗艙室 + 一兩個危險(水池/爆桶)」即可。浮島場地保留為另一張圖。

## 7. 版本路線（DoD）

### V0.8 — 事故報告雛形（**建議第一刀**；不需多人/伺服器/收容模式）
在現有 v2(搶獎盃 loop)結束時生成報告。
- 事故報告 UI + 名稱生成 + 等級 + 安全委員會評語 + 稱號 + 複製分享文字 + 10 種事故圖鑑。
- 複用 `recordDisaster`/`makeRunStory` + 補統計計數器(誤傷/自傷/損害%/連擊)。
- **DoD**：玩一局 → 跳出一份讀起來好笑、可一鍵複製的報告。找人看會不會想截圖。
- 驗的是：**這個包裝是否讓遊戲更有記憶點？**(成立 → 方向確立)

### V0.9 — 收容測試原型
- 失控法師 vs 收容員(對 AI)、實驗艙、穩定值/削弱、推進、3 秒關艙倒數、失敗過載爆炸。
- 報告支援收容類事故(反向收容/艙門短路/自行入艙…)。
- **DoD**：被推進艙→倒數→成功收容 or 過載爆炸；產生對應收容事故報告。

### V1.0 — 社群挑戰版
挑戰碼 / 每日事故重現 / 分享卡圖片(1200×630) / 本機排行 / 最後 8 秒回放 / 自訂條件。

### V1.1+ — 基地/地圖編輯器（遠期，先別碰）
玩家自建基地讓他人逃脫/收容 → 接非同步 PvP / 家園。

## 8. 風險與守則

- **範圍爆炸**：圖鑑/挑戰碼/每日/編輯器全是 V1.0+。**先把 V0.8 做到好笑再說**。
- **收容模式 vs 搶獎盃 loop**：兩者都只是「餵報告的引擎」，可並存或擇優；V0.8 不需選邊。
- **美術主軸**：浮島(室外) vs 實驗基地(室內)兩種調性 → 當成兩張不同場地，別互卡。
- **不對稱平衡**：1v1 先驗手感，1vN 之後再調。
