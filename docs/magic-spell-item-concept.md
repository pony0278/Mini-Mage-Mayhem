# 魔法事故報告：法術與道具系統設計稿

版本：v0.1  
定位：`魔法事故報告 / 收容測試` 的局內法術系統方向  
核心結論：**主玩法採用「場上法術卷軸 / 魔法道具，用完就丟棄」；少量保留 3 選 1 升級，作為事故異變。**

---

## 1. 設計結論

目前遊戲方向已經從單純的：

> 魔法師打怪 / 魔法大亂鬥

轉向更有特色的：

> 在失控的魔法實驗基地裡，玩家互相追捕、反抗、推進收容艙，最後產生一份荒謬的魔法事故報告。

因此，法術系統不應該以傳統 Roguelike 的「吃經驗、升級、3 選 1、合成終極法術」為主軸。

更適合的方向是：

> 玩家在場上搶奪各種不穩定的法術卷軸、魔法道具與實驗設備，用完即丟，過程中造成誤傷、反向收容、爆炸、短路、滑倒、封路等事故。

推薦比例：

| 系統來源 | 比例 | 功能 |
|---|---:|---|
| 場上卷軸 / 魔法道具 | 70% | 主要變化來源，製造混亂與笑料 |
| 角色初始法術傾向 | 20% | 給玩家基本身份與操作習慣 |
| 事故異變 3 選 1 | 10% | 少量成長感，讓局勢更失控 |

---

## 2. 為什麼不主打「吃經驗升級 3 選 1」？

### 優點

「吃經驗 → 升級 → 3 選 1 → 合成終極法術」很適合：

- 單人 PvE 生存遊戲
- Vampire Survivors 類型
- 長局成長
- Build 成形爽感
- 玩家期待後期大招成形

### 問題

但它不完全適合目前這款遊戲的新定位。

如果主玩法變成吃經驗升級，玩家注意力會被導向：

> 我要農經驗、湊 Build、升級火球傷害、合成大招。

這會讓遊戲逐漸變回比較普通的 Roguelike Arena。

而目前真正有辨識度的核心是：

> 把失控玩家推進實驗艙，中途各種法術與機關失控，最後產生好笑的事故報告。

所以主系統應該服務於：

- 追捕
- 逃跑
- 搶道具
- 推人
- 掙脫
- 誤傷
- 反向收容
- 場地事故
- 分享性結算

而不是服務於刷怪升級。

---

## 3. 為什麼「場上卷軸 / 魔法道具」更適合？

場上道具制更像：

> 實驗室裡到處都是危險設備，玩家邊跑邊亂拿來用。

這跟「魔法事故報告」的世界觀非常貼合。

### 優點

1. **更符合魔法實驗基地**  
   法術不是玩家穩定學會的技能，而是現場拿到的危險實驗用品。

2. **更容易產生笑料**  
   道具用錯、打歪、誤傷、觸發機關，都能變成事故報告素材。

3. **更適合追捕 / 逃跑玩法**  
   玩家會為了搶某個道具而改變路線，形成自然衝突。

4. **不容易滾雪球**  
   用完就丟，強道具不會永久堆疊，PvP 比較公平。

5. **畫面容易理解**  
   玩家看到地上有火球卷軸、冰霜瓶、傳送符，就知道那是可以搶的東西。

6. **每局都會不同**  
   道具位置、道具組合、場地機關、玩家行為會自然產生大量變化。

## 4. 核心遊戲循環

推薦核心循環如下：

```text
進場選一個基礎法術身份
↓
在實驗基地裡追捕 / 逃跑 / 反抗
↓
場上搶卷軸、撿魔法道具、啟動機關
↓
道具造成誤傷、短路、爆炸、滑倒、反向收容
↓
累積事故能量
↓
少量 3 選 1 事故異變，讓局勢更失控
↓
有人被推進實驗艙，或成功逃脫
↓
生成魔法事故報告
↓
被抓者以 UGC 造型變成展示艙樣本
```

---

## 5. 角色初始法術傾向

每位玩家進場前可以選擇一個「法術傾向」，而不是完整 Build。

這個選擇只決定玩家的基本風格，不應該讓玩家一開始就有太複雜的技能組。

| 法術傾向 | 玩法定位 | 適合角色 |
|---|---|---|
| 火系 | 爆炸、破壞、壓制 | 喜歡主動製造混亂的玩家 |
| 冰系 | 減速、滑倒、封路 | 喜歡控場與陷阱的玩家 |
| 雷系 | 麻痺、導電、短路 | 喜歡打斷與連鎖事故的玩家 |
| 風系 | 推人、位移、干擾 | 喜歡把人推進艙的玩家 |
| 土系 | 障礙、牆壁、震地 | 喜歡封鎖路線與保護艙門的玩家 |

### 初始技能數量建議

每位玩家進場最多：

- 1 個基礎法術
- 1 個通用動作，例如閃避、掙脫、短衝刺

不要一開始給太多技能，否則會影響新手理解。

## 6. 場上法術卷軸 / 魔法道具設計

### 道具設計原則

每個道具都應該同時具備三個功能：

1. **正常用途**：玩家知道可以拿來做什麼。
2. **事故用途**：玩家用錯或環境互動時會出現笑料。
3. **收容關聯**：能影響推進艙、逃跑、掙脫、艙門倒數。

---

## 7. 初期道具清單

| 道具名稱 | 正常用途 | 事故笑點 | 收容關聯 |
|---|---|---|---|
| 火球卷軸 | 發射爆炸火球 | 炸飛自己或炸壞艙門 | 可把受測體炸向艙口，也可能把他炸走 |
| 冰霜瓶 | 製造冰面 / 減速 | 自己滑進艙、隊友滑倒 | 可封鎖逃跑路線 |
| 雷擊棒 | 麻痺目標 | 打到導管造成短路 | 可打斷掙脫，也可能讓收容艙反向啟動 |
| 風壓手套 | 推人 / 推物件 | 把隊友推進艙 | 是最直接的收容工具 |
| 磁力石 | 吸引金屬物件 | 把陷阱吸到自己旁邊 | 可拉動鐵籠、艙門、金屬箱 |
| 傳送符 | 短距離位移或交換位置 | 反向收容 | 可把自己和對方位置互換 |
| 泡泡槍 | 暫時困住一名角色 | 困住錯的人 | 可短暫固定受測體，方便推進艙 |
| 超載核心 | 大範圍爆發 | 全員炸飛 | 可翻盤，但容易讓收容流程失控 |
| 黏液桶 | 製造黏地板 | 全員卡住 | 可拖慢逃跑者，也可能卡住收容員 |
| 反重力球 | 讓角色漂浮 | 大家撞天花板或被風吹走 | 可讓受測體更難控制方向 |

## 8. 道具稀有度與危險度

道具不只分強弱，也可以分「危險度」。

| 危險度 | 說明 | 範例 |
|---|---|---|
| 安全級 | 效果直覺，不太會反噬 | 小火球、短推風掌 |
| 注意級 | 可能造成誤傷 | 冰霜瓶、泡泡槍 |
| 危險級 | 容易觸發場地事故 | 雷擊棒、磁力石 |
| 禁用級 | 強力但高度失控 | 超載核心、反重力球 |

這樣可以讓道具自然對應事故報告的等級。

例如：

> 本次事故涉及 2 件危險級道具與 1 件禁用級道具，事故等級提升為 S。

## 9. 事故能量系統

不要叫經驗值，因為「經驗值」會讓玩家聯想到傳統升級打怪。

推薦名稱：

- 事故能量
- 實驗數據
- 魔力殘渣
- 失控能量
- 過載值
- 異常指數

### 累積方式

事故能量不應該只靠擊殺取得，而是靠製造有趣事件取得。

| 行為 | 事故能量 |
|---|---:|
| 成功推動受測體靠近收容艙 | +少量 |
| 成功掙脫收容 | +少量 |
| 用道具造成多人擊飛 | +中量 |
| 觸發元素連鎖 | +中量 |
| 誤傷隊友 | +少量，並記錄事故 |
| 反向收容 | +大量 |
| 艙門倒數被打斷 | +中量 |
| 同時影響 3 名以上角色 | +大量 |
| 自己被自己的法術害到 | +少量，但會解鎖事故報告條目 |

這會鼓勵玩家製造混亂，而不是只追求擊殺。

## 10. 事故異變 3 選 1

3 選 1 可以保留，但不要作為主成長。

它應該是：

> 玩家累積足夠事故能量後，基地安全系統偵測到異常，給予一次不穩定的魔法異變。

一局最多出現 1～2 次即可。

### 設計原則

事故異變不應該是純強化，而是：

> 變強，但更危險。

例如：

| 異變名稱 | 效果 | 代價 |
|---|---|---|
| 爆燃增幅 | 火球半徑 +20% | 自傷範圍也增加 |
| 脆冰擴散 | 冰面持續時間 +2 秒 | 自己也更容易滑倒 |
| 導電強化 | 雷擊可多彈跳 1 次 | 可能彈到收容艙導線 |
| 風壓過載 | 推人距離增加 | 近距離會把自己往後彈 |
| 緊急築牆 | 土牆更厚 | 可能封住自己退路 |
| 泡泡增壓 | 泡泡困住更久 | 泡泡破裂時會爆開 |
| 傳送不穩定 | 傳送距離增加 | 落點有小幅偏移 |
| 事故吸引體質 | 道具刷新更靠近自己 | 危險道具也更常在附近出現 |

這種升級方式會強化遊戲主題：

> 你不是變成穩定的大法師，而是變成更危險的事故源。

---

## 11. 終極法術設計：不是大招，而是過載事故

不建議用傳統：

```text
吃經驗 → 合成 → 穩定終極法術
```

推薦改成：

```text
多次使用同元素道具 / 累積過載值
↓
觸發元素過載
↓
產生強力但不穩定的終極事故
```

## 12. 五系終極事故範例

### 火系終極：實驗室爆燃

效果：

- 大範圍火焰爆炸
- 可瞬間清場或炸開障礙
- 對收容員與受測體都有效

事故風險：

- 可能破壞收容艙門
- 可能炸飛施放者
- 可能引燃其他道具

事故報告範例：

> 受測者試圖以火焰清除障礙，結果同時清除了障礙、收容員與半個實驗室。

---

### 冰系終極：全域冷卻失控

效果：

- 整個房間降溫
- 地面結冰
- 所有人移動變滑

事故風險：

- 逃跑者可能自己滑進艙
- 收容員可能集體滑倒
- 艙門可能因結冰延遲關閉

事故報告範例：

> 本次收容並非計畫成果，而是受測體在冰面上自行滑入收容艙。

---

### 雷系終極：導管連鎖短路

效果：

- 雷電沿基地導線連鎖跳躍
- 麻痺多名角色
- 暫時關閉部分機關

事故風險：

- 收容艙可能短路
- 艙門可能反向啟動
- 可能電到自己或隊友

事故報告範例：

> 技術上來說，收容程序成功啟動。問題是啟動方向錯誤。

---

### 風系終極：壓力艙暴走

效果：

- 產生強風流
- 大範圍推動角色與道具
- 可快速把受測體吹向艙口

事故風險：

- 可能把收容員吹進艙
- 可能把道具吹到不可預期位置
- 可能造成連鎖碰撞

事故報告範例：

> 收容員成功製造了通往收容艙的強風通道，並第一個抵達目的地。

---

### 土系終極：緊急隔離牆

效果：

- 生成多面隔離牆
- 封鎖逃跑路線
- 保護收容艙入口

事故風險：

- 可能把自己關在錯的房間
- 可能阻擋隊友追捕
- 可能讓受測體安全逃走

事故報告範例：

> 本次隔離措施確實成功隔離了危險目標。遺憾的是，危險目標位於牆的外側。

## 13. 元素互動與事故反應

元素反應應該服務於「事故」，不要只做成傷害加成。

| 元素組合 | 反應 | 事故效果 |
|---|---|---|
| 火 + 冰 | 蒸氣爆破 | 視線遮蔽、近距離擊飛 |
| 雷 + 水 / 蒸氣 | 導電 | 電到多人，也可能電到自己 |
| 火 + 風 | 火焰旋風 | 持續移動的火焰區域 |
| 冰 + 風 | 暴風雪 | 大範圍滑倒與減速 |
| 土 + 火 | 熔岩裂縫 | 地面危險區，可能封死退路 |
| 土 + 雷 | 磁暴震擊 | 吸引金屬物件與陷阱 |
| 風 + 雷 | 閃電亂流 | 雷擊方向不穩定，可能偏轉 |

這些反應的目的不是讓玩家算最強 DPS，而是製造：

- 誤傷
- 反彈
- 場地變化
- 艙門異常
- 逃跑路線改變
- 事故報告素材

---

## 14. 收容艙與法術的關係

收容艙不是單純終點，而是整場遊戲的核心互動物件。

法術與道具都應該能影響收容艙。

| 互動 | 效果 |
|---|---|
| 火焰爆炸打到艙門 | 艙門暫時卡住或過熱 |
| 冰霜打到艙門 | 艙門關閉速度變慢 |
| 雷擊打到艙體 | 短路、倒數暫停、反向吸入 |
| 風壓打到艙口 | 增加吸入方向推力 |
| 土牆生成在艙口 | 阻擋收容或形成牢籠 |
| 磁力石靠近艙門 | 拉動金屬艙門，可能提前關閉 |
| 超載核心爆炸 | 艙體故障，全員彈飛 |

這能讓每一場收容都不只是「把人推到點位」，而是變成一個會失控的程序。

## 15. 勝負與事故評價

遊戲不應該只判斷勝負，還要判斷事故等級。

### 收容員勝利

條件：

- 成功把受測體推進收容艙
- 艙門倒數完成
- 受測體被暫時冷卻 / 收容

### 受測體勝利

條件：

- 撐過時間
- 逃出基地
- 破壞收容艙
- 讓收容員被反向收容

### 事故等級

| 等級 | 條件範例 |
|---|---|
| E | 普通收容或普通逃脫 |
| D | 有少量誤傷或道具失控 |
| C | 場地出現明顯破壞 |
| B | 觸發元素連鎖或艙門異常 |
| A | 多人被同一事故波及 |
| S | 反向收容、基地嚴重損壞、爆笑翻盤 |
| S+ | 勝負結果極度荒謬，例如受測體自行滑入艙、收容員被關進艙仍判定任務完成 |

## 16. 魔法事故報告如何引用法術系統

結算報告應該自動讀取本局的關鍵事件。

### 報告資料來源

- 使用最多的道具
- 造成最大事故的道具
- 最後觸發的元素反應
- 是否發生反向收容
- 是否有人自己害死自己
- 是否有艙門短路
- 是否有隊友誤傷
- 是否有基地損害
- 被收容者的 UGC 造型
- 被收容時的姿勢

### 報告範例

```text
魔法事故報告 #118

事故名稱：反向收容與雷擊短路事件
事故等級：S

受測體：南瓜頭火球法師
收容員：3 名
主要涉案道具：雷擊棒、風壓手套、冰霜瓶

事故摘要：
收容員成功將受測體推至收容艙門口。
然而艙門在關閉前遭雷擊棒誤擊，導致收容程序反向啟動。
一名收容員被吸入艙內，受測體趁亂使用冰霜瓶逃離。
逃離途中又因冰面滑倒，二度撞回艙門前。

結果：
收容失敗
反向收容：1 次
基地損害：72%
誤傷隊友：3 次

安全委員會評語：
「技術上來說，有人被成功收容了。只是收錯人。」
```

## 17. V0.8 最小可行版本

第一版不要做太多法術與升級，重點是驗證：

> 場上道具 + 收容艙 + 推人 + 事故報告是否好笑。

### V0.8 必做

1. 一張小型實驗室地圖
2. 一個收容艙
3. 1 名受測體 AI 或玩家
4. 1 名收容員玩家
5. 推人 / 擊飛 / 掙脫機制
6. 5 個場上道具
7. 3 種事故判定
8. 一份結算事故報告

### V0.8 道具建議

先做 5 個最能驗證核心的道具：

| 道具 | 為什麼先做 |
|---|---|
| 風壓手套 | 最直接驗證「推進艙」是否好玩 |
| 冰霜瓶 | 最容易產生滑倒與自投羅網笑料 |
| 雷擊棒 | 可測試艙門短路與反向收容 |

## 19. 最重要的設計守則

### 守則 1：法術不是單純武器，而是事故來源

每個法術都應該能：

- 幫助玩家
- 害到玩家
- 影響場地
- 影響收容艙
- 產生事故報告素材

---

### 守則 2：升級不是純變強，而是更不穩定

如果有 3 選 1，它應該像：

> 魔法異變 / 實驗失控 / 過載副作用

不要像：

> 傷害 +10%、冷卻 -10%、暴擊 +5%。

---

### 守則 3：勝負不是唯一目標

玩家應該同時追求：

- 贏
- 製造 S 級事故
- 讓朋友出糗
- 把對方變成展示品
- 解鎖新的事故報告

---

### 守則 4：每場都要能說成一個故事

好的局內事件應該能被玩家講出來：

> 我快把他推進艙了，結果他用傳送符跟我換位置，我自己被關進去。

這種句子比數值成長更重要。

---

## 20. 一句話總結

這款遊戲的法術系統不應該是傳統 RPG 法術樹，而應該是：

> **一套散落在實驗基地裡的危險魔法器材系統。玩家邊追捕、邊逃跑、邊亂用道具，最後把整場混亂寫成一份魔法事故報告。**

核心不是：

> 我升級出了最強法術。

而是：

> 我撿到一個危險卷軸，結果把對手、隊友、自己和收容艙一起搞壞了。

這才是「魔法事故報告」真正的獨特性。
