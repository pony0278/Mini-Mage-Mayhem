# v2 地板化學：元素反應系統（設計藍圖）

> 狀態：**設計定稿、尚未實作**（等核心「好笑」驗穩才動工——見 §7 紀律）。
> 定位：這是 [v2-spec-elements.md](v2-spec-elements.md) 說的**模型 ③「地圖撿元素」變化層**的地板端規則。
> 主軸不變：元素＝**空間動詞**（把人送進艙/打暈好抓的混亂層），不是 DPS。

---

## 0. 一句話

**一格地板只有一個狀態(取代,不疊層)。** 丟一個元素上去,要嘛觸發**招牌反應** →
變成**另一個單一狀態**,要嘛**取代**舊底料,要嘛**無效**。融合不在法術欄、在地板上。

> 為什麼是「取代不疊層」:**可讀性是 PvP 的硬天花板**(spec-elements §1)。
> 疊層 = N×N 組合爆炸,對手一眼讀不懂 → 砍。一格一狀態 = 永遠可秒讀。

---

## 1. 大原則:一格一狀態 + 招牌反應(取代,非疊層)

- **地板是一台狀態機**,每格只落在一個狀態(見 §3)。
- 元素落到某格 → 查「(現狀態 × 元素)」:
  1. **有招牌反應** → 轉成新的**單一**狀態(可能附一次性事件,如毒爆)。
  2. **可當底料** → **新底料取代舊底料**(油→水:水覆蓋)。
  3. **都不是** → **無效**(no-op)。
- **絕不出現「油+冰+毒」三層疊在同一格**。要嘛反應成一個新狀態,要嘛後者取代前者。

> 心智模型:元素是**轉場箭頭**,不是**圖層**。地板永遠只有「現在是什麼」一個答案。

## 2. 招牌反應表(~6 條,複用單機既有原語)

複用單機 `sim.js` 已驗證的反應原語(火燃油/草、雷導水、火引爆毒),只是搬到 PvP 地板語意:

| # | 觸發 | 現狀態 | 結果狀態 | 演出 / 效果 |
|---|---|---|---|---|
| R1 | 🔥 火 | 油 / 草 | **火海**(fire) | 沿**相連的油**擴散;站上去燒人、擋路 |
| R2 | ⚡ 雷 | 水 | **電水**(charged_water) | 踩進去 → 自電硬直(stun),經典自作自受 |
| R3 | 🔥 火 ↔ ☠️ 毒 | 毒(遇火) / 火(遇毒) | **清空**(clean) + 一次性毒爆 | 一次性 AoE 爆炸,炸完該格歸零 |
| R4 | ❄️ 冰 | 火 | **水**(water) | 熄滅 → 留下水(可再被 R2 電化) |
| R5 | ❄️ 冰 | 油 / 水 | **冰面**(ice) | 變滑 → 滑步/失控滑進艙/危險區 |
| R6 | 🌀 風 | (不改地板) | — | **不改地板**;吹動火/毒雲、把**人**送進上面任一狀態格 |

**未列的配對** = 走 §1 的規則 2/3(新底料取代 or 無效)。表只列**會產生新意義**的組合。

> 每條反應都通過那把尺(spec-elements §3):**改變空間 → 保留**;純扣血 → 降級。
> 毒的慢性扣血是**慌張計時器**(逼對手離開毒區),不是主要傷害來源。

## 3. 地板狀態機(格子狀態 + 轉場)

**狀態集**:`clean`(乾淨) / `oil`(油) / `water`(水) / `ice`(冰面) / `poison`(毒) / `fire`(火海) / `charged_water`(電水)。

轉場(箭頭 = 對該格丟某元素):

```
clean ──油──▶ oil ──🔥──▶ fire ──❄️──▶ water ──⚡──▶ charged_water
  │            │  ╲                        ▲              │
  │            │   ╲❄️                     │❄️(熄)        │(自然衰退)
  │            │    ▼                       │              ▼
  │            └──▶ ice ◀──────────────────┘            water
  │
  ├──水──▶ water ──❄️──▶ ice
  ├──毒──▶ poison ──🔥──▶ (毒爆) ──▶ clean
  └──(火/毒 相遇)──▶ (毒爆) ──▶ clean

fire  ──(相連 oil)──▶ 火海沿油擴散(R1)
風    ──▶ 不改格子狀態,只搬「人 / 雲」
所有狀態 ──(計時)──▶ 自然衰退回 clean(火海燒完、冰融、毒散)
```

- **底料互換**:`oil ⇄ water`(後丟者取代;水覆蓋油 = 撲滅可燃性,但不等於清潔)。
- **一次性事件**(毒爆)不是狀態,是「轉場時放一次的 AoE」,放完該格回 `clean`。
- **自然衰退**:每個危險狀態都有壽命 → 回 `clean`(避免地圖被永久污染鎖死)。

### 3.1 衰退壽命(起始值,之後實測微調)

**統馭原則:危險度 ↔ 壽命成反比。** 主動殺傷活越短(不能 camp);鋪陳/佈場活越久(才有設連段、逼位移的時間)。

| 組 | 狀態 | 起始壽命 | 理由 |
|---|---|---|---|
| 底料/鋪陳(長) | oil、water | **~10s** | 錨定「自 combo 跑動窗口」:油必須活得比「鋪→跑去撿火→回來」久一點點,否則自接永遠來不及 |
| 佈場/控制(中) | poison、ice | **~8s** | 逼位移要夠久才有壓力;毒 1 發活 8s 剛好值那 1 次 |
| 主動殺傷(短) | fire、charged_water | **~4s(電荷)** | 火海不能當永久牆卡場;電荷 4s 後**退回水**(見下),不是整格消失 |

三個機制定案(決定衰退怎麼跑):

- **火海 = 滾動的浪**:每格火各自短命(~4s)+ 沿相連油傳播 → 火像波浪從點火處滾過油田、燒完即滅,是**會移動的危險**(好讀、戲劇、天然不 camp),**不是**整片同時燒的靜態牆。
- **電水衰退 → 變回水(可反覆充電)**:電荷 4s 後退回 `water`(**保留水的剩餘壽命**),可被新的雷再充。物理直覺(電荷散、水還在)、濕地變持續戰略資產。
  - **鐵則:充電不刷新水的底料時鐘。** 水底下那個 ~10s 衰退時鐘一直在跑,充電只是疊 4s 電荷 overlay,**不重置**水壽命;水自身 10s 到期則整格消失(不管中途充過幾次)。否則「充電→全新水→再充→永不衰退」會把整圖鎖成不死污染。
  - **不會 camp**:雷是 1 次數道具,每充一次就用掉手上唯一那發雷 → 要再充得跑去撿下一發 → 跑動成本天然限制頻率(一片水壽命內大概只夠充 1–2 次)。
  - **被新元素蓋過**:丟油(油水互換)、丟冰(R5 冰面)都會蓋掉電荷/水,走 §1 底料取代規則。
- **可讀性**:每個危險狀態**最後 ~1s 閃爍預警**再消失(純演出,實作順手加)。

## 4. 兩個已答情境(當實作驗收案例)

使用者提的兩個複合情境,結論即上表:

1. **「油地板燃燒中,另一人丟毒黏球到火上」**
   → `fire`(由 oil 點燃) 遇 **毒** = **R3 毒爆**:一次性 AoE 炸開、該格清空。
   *不是*「毒疊在火上」,是火毒相遇 → 一次性大事件 → 歸零。

2. **「地上是油,有人撿到冰霜瓶丟上去」**
   → `oil` 遇 **冰** = **R5 冰面**:油被**轉化成滑面**。
   關鍵:**不是單純取消/覆蓋**,而是把「可燃陷阱」變成「滑動陷阱」——
   性質改變(可燃 → 濕滑),仍是個能坑人的空間動詞。

> 這兩例正好示範兩種路徑:R3 = 招牌反應(新事件+歸零)、R5 = 招牌反應(性質轉化)。
> 都**不疊層**,都產生**一個新的單一狀態/事件**。

## 5. 已拍板的決策

1. **元素災難 = 工具與途徑**,把人**送進艙 / 打暈好抓**(混亂層)。
   與收容勝利**不衝突**:環境 = 擊殺 = 收容;元素災難是**餵給收容的前置**,不是平行傷害系統。
2. **觸發 = 自然定時 + 玩家觸發 混合**:場地會自己漏(定時災難源),玩家也能主動點燃/鋪設引爆。
3. **複合性靠地板、不靠法術欄**:玩家 A 鋪油 → 玩家 B 撿火棒點燃 → 玩家 C 撿毒球丟火上毒爆。
   多人各丟一手,地板把它們**串成一條因果**(有作者、有自作自受、有延遲引爆)。

## 7. 紀律(別越界)

- 這是 spec-elements 的**模型 ③ 變化層**:**現在設計、先不建**,等核心「3–4 人笑出聲」驗穩(規格 A)再上。
- 每條反應過尺:**改變空間 = 留;純 DPS = 砍**。反例:「火海傷害 +30%」。正例:「火點燃相連油成火海擋路」。
- **狀態數守恆**:狀態集控制在 ~7 個以內;要加新元素先問「會不會逼我加一排新狀態 → 破壞可讀性」。
- **不疊層是鐵則**:任何時候想「讓兩個效果同時在一格生效」,停——改成招牌反應(一個新狀態)或取代。

## 9. 元素 × 道具對照(建模藍圖)

核心邏輯:**七個元素 = 七種不重疊的投放原型**,動畫骨架才拉得開;命名維持「廢料魔法」調性。
每個道具的「連段角色」= 它在 §2 招牌反應裡的位置(道具 = 反應的載體)。

| 元素 | 定稿道具 | 投放原型 | 手感定位 | 魔法感重點 | 連段角色(→ §2) |
|---|---|---|---|---|---|
| **火** | 過熱核心噴火帽 | 噴流／衝撞 | 低頭前噴、近中距離火柱 | 藍白核心火、帽緣符文環、腳下法陣;火要「有秩序」 | 引燃油膜(R1) |
| **雷** | 魔導粗電線(電鞭) | 牽引 | 甩出纏住→灌電→拉近,最有記憶點 | 電流沿線流動符文、拉近時電弧 | 濕地導電加傷(R2) |
| **水** | 魔法水桶・壓 | 砸壓(由上而下領域) | 懸停蓄力→極快砸下,大開大合 | 流體**不潰散**、懸停停頓、發光濕痕 | 產出濕地接雷(R2) |
| **毒** | 紫色污染廢液桶 | 放置領域 | 倒地形成毒池/毒霧,拖延型 | 廢液發光漂浮、紫符文尾跡、毒霧浮光 | 火↔毒毒爆(R3)、控場逼位移 |
| **冰** | 冰霜瓶(地面冰刺) | 投擲 | 砸地後成範圍冰面 | 冰面竄出的霜白符文 | 冰面(R5),滑動逼位移 |
| **油** | 事故潤滑桶(油膜) | 投擲 | 潑油膜、地板變滑、可被引燃 | 油面反光符文、被點燃瞬間能量擴散 | **連段樞紐**(油+火、油+雷) |
| **風** | 風壓手套 | 近戰衝擊 | 出拳送氣浪、擊退、反彈投射物 | 氣流成形旋渦紋、拳套符文亮起 | 反彈、送人進艙(R6) |

**投放原型無重疊**:噴流(火)／牽引(雷)／砸壓(水)／放置(毒)／湧現(冰)／投擲(油)／近戰(風)——
七個各一種,骨架全不同,杜絕「換色而已」。**投擲只留油一個**(刻意的特例;油天生是連段引信)。

**貫穿原則**:元素要看起來**被意志控制、違反物理**(水該散不散、火該亂不亂、氣流成形)+ 發光符文與腳下法陣
→ 同時保住廢料喜感與魔法感,而非「意外在燒/在潑」。

### 9.1 四項拍板定案(2026-07 討論)

1. **土 = deferred(不刪,擱著)**。目前遊戲不出現;未來 netcode 成熟可能回來。
   砍的理由:造牆/裂地 = 唯一要**改幾何結構**的元素(每幀同步地形 + 重算碰撞),netcode 最貴;
   場地是固定回收中心,瓶頸已由場景設計死,主動造牆收益有限。油+水升為正式道具剛好補齊底料生產者,
   七件套自成閉環,土反而是唯一不進地板化學的孤兒。**送人進艙靠推/滑/送(風冰水),不靠擋(土)。**

2. **元素道具 vs 工具道具:概念分兩類、資料同一張表**(靠 `ITEM_SPEC.kind` 區分,見 item-cast 文件)。
   - **元素道具(這 7 個)**:走本表規則(原型不重疊、有連段角色、吃地板化學、元素色+符文+法陣)。
   - **工具道具**:不吃地板化學、自成一格、廢料科技風。**MVP 只放傳送一個**(`kind:'mobility'`、`whileDisabled:true`)。

3. **水 = 造濕地 + 砸中擊倒/暈一下(好抓)**。直擊傷害只是佐料,**擊殺理由是「濕地 → 接雷電水」**;
   砸中的**短擊倒**是空間動詞(剝奪移動、好抓送進艙),一魚兩吃(餵地板化學 **又** 服務「打暈好抓」)。
   刻意避免把「壓扁」做成一發打死的 DPS(Path-A 陷阱)。

4. **火/雷 MVP 先退化成單發**,塞現有單 impact 骨架;channel/多階段骨架驗穩後再擴(見 item-cast §「投放原型 × 骨架」)。
   - 火 MVP = 一發短噴(一個 impact 點燃前方油/造小塊火),**先不做持續火柱**。
   - 雷 MVP = 一發電擊(一個 impact,濕地加傷/自電),**先不做纏住拉近**。
   - 理由:火的遊戲價值在「點燃油成火海」(地板化學給的,不靠火柱持續);雷的記憶點「拉近」最貴,壓後面。

### 9.1.1 道具次數(3/2/1 三階)+ 自 combo

**次數 = 角色說明書**:數字本身教玩家這道具是哪種角色(撿到「油×3」直覺=拿來鋪的;「雷×1」=珍貴一擊)。
撿取時 `f.itemUses = ITEM_SPEC[type].uses`。

| 階 | 角色 | 次數 | 道具 | 理由 |
|---|---|---|---|---|
| A 鋪陳/推送 | 用量大、單發弱、靠反覆使用 | **3** | 油、風 | 油要潑夠多片才成連段;風是核心送人動詞要能反覆轟 |
| B 觸發/中堅 | 單發有份量、非終結 | **2** | 火、水 | 點燃/砸壓是關鍵一手,給兩發容錯 |
| C 終結/滯留/逃脫 | 強終結 / 一發夠久 / 保命 | **1** | 雷、冰、毒、傳送 | 雷=最強終結;冰毒=一發在地板留 8s(靠衰退撐時長);傳送=保命稀有 |

**1v1 自 combo = 允許,但有跑動成本**:一次只持有一個 `f.item`,自接「油→火」要「鋪油 → 跑去撿火 → 回來點」。
油壽命(~10s,見 §3.1)刻意設定成**撐得過這趟來回但很趕**,對手有機會離開油區 → 連段要移動+時間成本,不是一鍵。
(未來多人時自然變成多人各丟一手、地板串因果的原始願景。)

### 9.2 建模／落地順序(給建模用)

七個原型全模是大量工。表自身指出**油是樞紐**,故最小可玩迴圈只要三件套:

1. **第一批 · 油 + 火 + 風**(投擲鋪面 → 引燃成火海 → 把人轟進火海)。
   「鋪地板 → 點燃 → 送人進去」完整因果,能玩、能笑、能驗 fun。**先做這批。**
2. **第二批 · 水 + 雷**(造濕地 → 導電電水)。雷是最有記憶點但動畫+機制最貴,壓到第二批剛好。
3. **第三批 · 冰 + 毒**(控場、逼位移、補齊 R5/R3)。

每加一批 = 一組能玩的新連段,建模不用一次背七個。

## 10. 元素站與洩漏節奏(場地災難層)

四角元素站(fire/frost/poison/lightning,已在 `render-lab.js` 建好)= 回收中心的**洩漏事故源**。
與道具的關係是**不同尺度、零重複**:

| | 道具(手上的) | 元素站(場地的) |
|---|---|---|
| 尺度 | 小、精準、玩家瞄準 | 大、範圍、環境級 |
| 控制 | 你決定何時何地 | 場地自己漏(拉桿啟動後) |
| 心智 | 「**我**在這裡製造危險」 | 「**這個角落**很危險」 |
| 敘事 | 撿到的廢料魔法 | 回收中心的洩漏事故(場景主題) |

**一句話:道具是手術刀,元素站是定時炸彈。** 站是**同一批元素的環境大尺度版**——把人騙進即將噴發的角落,或自己被吞。

### 10.1 觸發:總開關(緊急拉桿),不是攻擊個別站

- **開局是關的**:每局開場**平靜**(純比手感/對拳的試探期),沒有環境危險。
- **任一玩家打「總開關」**(場地上的緊急拉桿)→ 啟動四站輪噴循環。
- **單向、不可反關**(先試單向):拉了就開到這階段結束;對手不能關掉,雙方一起面對(對稱風險 = 拉桿是個真決策,可能反噬自己)。
- **敘事**:拉桿=設施收容失控;可接規格 F 三階段收容升級(拉桿=把事故升一階)。
- **「攻擊個別站體提前引爆」= 之後的進階層,MVP 不做**(避免兩套觸發並存)。

### 10.2 節奏:錯開輪噴 + 預警

- **四站輪流噴、不同時**:~**30s/站**、四站**錯開 10s** → 平均每 ~10s 一個角落進危險。刻意放慢——環境是底噪,太頻繁會把玩家煩死。
- **噴前 ~3s 預警**:該站發光/震動 + 地面浮現警示圈。這 3s 是**戰術窗口**:看到毒角要噴,趁機把對手逼進去(有預警=蠢死法演出;無預警=隨機死)。
- **噴發產出**:該站元素的**一大片地板狀態**(火站→大片火海、冰站→冰面、毒站→毒霧),落地即交給地板化學(火站旁有油→連鎖更大火海)。

### 10.3 雷站 = 原始電弧(特例)

雷的地板效果是「充電水」,但雷站噴在乾地無水可充。故雷站走 **(b) 原始電弧**:發電機漏電,直接一片**電擊硬直區**(不需要水)。
- 讓四站存在感對等(火燒/冰滑/毒醺/電麻,當場都能坑人)。
- **手上的雷道具仍走「充水」精準路線** → 站=raw、道具=需鋪陳,又剛好分了尺度。
- **暫不做**「噴發朝中央推(送進艙)」加分項(先簡單、中性擊退)。

## 11. 道具刷新 / 搶點(model ③ 地圖撿元素)

**定位:道具搶點是「次要爭奪」,餵給「主要爭奪(送人進中央清運口)」。** 別搶戲成「搶道具遊戲」。
空間流:**去外圍撿工具 → 打回中央 → 送進艙**(呼應 §6 四色分揀路線把災難往中央輸送)。

### 11.1 刷新權重 = 鏡射次數三階

道具的**刷新頻率 / 全場同時數**對應它的角色,與 3/2/1 次數系統疊加(稀有度 × 次數 雙重):

| 層 | 稀疏程度 | 全場同時 | 重刷(起始) | 道具 | 效果 |
|---|---|---|---|---|---|
| 基礎(鋪陳) | **保底** | 1–2 | ~10s | 油、風 | 你**永遠**撿得到 → 核心「鋪地板→送人」迴圈不斷氣 |
| 觸發(中堅) | 稀疏 | 0–1 | ~15s | 火、水 | 「點燃/砸壓」是要去找/搶的一手 → 火成焦點爭奪 |
| 終結(spike) | **很稀有** | **全場最多 1**(共用稀有槽) | ~30s+ | 雷、冰、毒、傳送 | 撿到雷=大事件,雙方衝同點互炸 |

**稀疏「分層」不是一視同仁**:整體感覺稀疏(常常全場只 2–3 個、稀有槽最多 1),但**油/風保底 → 沒有死時間**
(避免「用完道具→什麼都沒有→只能站著互揍」的空窗爛狀況)。玩家撿到火會興奮、撿到雷會尖叫。

### 11.2 刷新形式:MVP 固定刷新台 → 之後升級輸送帶

- **(a) 固定刷新台(MVP)**:外圍 ~4–5 個固定台,撿走後依稀有度重刷、錯開 → 同時刻只有部分亮著,得去搶亮的那個。經典 arena shooter、好學、好實作、位置可預判。
- **(b) 輸送帶送出(之後主題升級)**:回收中心沿 `conveyorBelt`(render-lab 已有 prop)把廢料魔法送出來,從帶上抓。**額外價值:看得到道具正在送來 = 預警**(同元素站噴發預警的可讀性),能提前卡位。實作較複雜(移動道具、抓取時機),故當升級層,非 MVP。

## 12. 不穩定魔力廢料桶(Unstable Arcane Waste Barrel)

> 狀態:**步驟 A + B 已實作**(受擊/被丟升壓→1s→擊飛+種地板+元素充能+發光 telegraph;可推/撿/丟+撞擊引爆+在手上爆)。桶模型仍佔位方塊。

**取代舊「爆桶」。** 舊爆桶(火藥桶梗、4 個對稱固定點、走近自動引爆)在三軸上跟主題脫鉤:
fiction 走錯棚(魔法事故現場不該有火藥桶)、機制跟元素站重疊(都是環境 AoE 送人)、擺位跟場地無關。
重定位成**回收中心的不穩定廢料桶**後,它同時吃三套系統 → 從「該刪的異物」翻成「該留的核心玩具」:
**carry/throw 動詞 × 地板化學 × 元素系統**,在一個物件上交會。

### 12.1 行為規格

1. **可推 / 撿 / 丟**:複用既有 carry/throw 動詞(玩家已在抓人丟人進艙)。丟桶 vs 丟人是**佔手的抉擇**。
2. **受攻擊 or 被丟出 → 開始升壓**(取代舊的「走近自動引爆」)。**被動近距引爆已拿掉** → 必須打它/丟它,更有意圖、少誤觸。
3. **升壓 ~1 秒後爆炸**(比舊 0.5s 長 → 給反制窗口:跑開 / 抓起來丟回去 = 手榴彈)。
4. **大範圍擊飛**(= 送人進艙的酬載;沿用舊 explodeBarrel 的 AoE)。
5. **爆完留一塊污染地面**(呼叫 `stampElement` 種下地板狀態 → 每次爆炸都接地板化學連段)。
6. **可被元素改變效果**:最後打中它的元素決定**爆的種類 + 留下的地板**(水打→大水花+地板 water、火打→火爆+burnt…)。

### 12.2 關鍵洞察:第 5、6 是同一個機制

- **被某元素充能過** → 爆該元素 + 種該元素地板(= 第 6)。
- **沒被充能(預設)** → 爆「野生魔力」+ **隨機**污染(油/水/毒…)(= 第 5)。

**5 是 6 的「未程式化預設」。** 實作:桶記一個 `charge`(最後命中的元素);爆炸時 `stampElement(charge ?? 隨機)`。
統一、且 `stampElement`(第三刀)已現成。

### 12.3 三項拍板細節

1. **升壓發光 telegraph(可讀性)**:升壓 1 秒期間桶**發出 charge 的元素色光**(水藍/火橘/野生紫)。
   對手一眼讀出「這是水彈」→ 隨機也變可讀,升壓發光本身就是好 telegraph。**沒這條隨機會變成不講理的死。**
2. **MVP 只有冰能充能**:火/水道具還沒建模 → MVP 桶大多是**野生隨機爆 + 冰充能**;等 oil/fire/water 道具或元素站上線,充能才全面展開。**架構先讀 `charge`,表現隨元素源豐富**(隨機爆本身好玩又貼「事故」主題)。
3. **拿掉被動近距引爆**:改成攻擊/被丟才升壓(見 12.1-2)。

### 12.4 可建性(純程式,不需建模)

| 部件 | 複用 |
|---|---|
| 污染地板 | `stampElement`(第三刀)✅ |
| 爆炸 AoE(擊飛+削穩定值) | 舊 `explodeBarrel` 邏輯 ✅ |
| 抓/丟 | 複用 carry/throw(桶非 fighter → 一個「可搬物件」小平行結構,中等工) |
| 充能 | 桶加 `charge` 欄位;wind/ice/(未來 fire/water)命中時寫入 |

**機制純程式、桶的模型之後再做**(先佔位方塊也能玩)。做完立刻有價值:**在 oil 道具建模好前,打爆一個野生桶就能種出隨機地板(油/水/毒)→ 現在就能測地板化學連段。**

### 12.5 擺位 / 數量 / 頻率(定案)

**羅盤分區**:多個生成系統各占場地一個方位、互不重疊(解決「桶跟補給台位置撞」的擔心,也讓場地可讀):

```
   ◆站(角)      ▓ 補給台N (480,140)      ◆站(角)

     桶W ───────  ◎清運口 (480,320) ─────── 桶E
   (200,320)                              (760,320)

   ◆站(角)      ▓ 補給台S (480,500)      ◆站(角)
```

| 方位 | 系統 | 座標 |
|---|---|---|
| 中央 | 清運口(主戰場) | (480,320) |
| 南北中線 | 補給台(§11 撿道具) | (480,140)、(480,500) |
| **東西中線** | **廢料桶** | **(200,320)、(760,320)** |
| 四角 | 元素站(§10,之後做) | 角 |

**定案值**:

| 項 | 舊 | 定案 | 理由 |
|---|---|---|---|
| 數量 | 4 | **2(東西對稱)** | 減量=底噪更淡、每次爆更有意義;桶會移動不需多;1v1 對稱公平 |
| 位置 | 內矩形 4 點 | **(200,320)(760,320)** | 東西中線 → 跟補給台(南北,x=480)不同軸、避開角(站)與中(艙),零重疊 |
| 頻率 | respawn 6s | **維持 6s** | 桶變少後爆炸間隔自然拉長 = 更好 |

**桶可推/可丟 → 這些座標只是「新桶誕生處」**(初始不打架、不擋別的系統),不是戰鬥固定發生處——玩家會把桶搬去清運口旁炸。
(1 個中央焦點桶=更強「衝同點互炸」張力但 1v1 對稱性差,暫不採;之後想加再試。)
