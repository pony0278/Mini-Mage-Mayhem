# v2 地板化學：元素反應系統（設計藍圖）

> 狀態：**設計定稿、尚未實作**（等核心「好笑」驗穩才動工——見 §7 紀律）。
> 定位：這是 [v2-spec-elements.md](v2-spec-elements.md) 說的**模型 ③「地圖撿元素」變化層**的地板端規則。
> 主軸不變：元素＝**空間動詞**（把人送進艙/打暈好抓的混亂層），不是 DPS。

---

## 0. 一句話

**一格地板只有一個狀態(取代,不疊層)。** 丟一個元素上去,要嘛觸發**招牌反應** →
變成**另一個單一狀態**,要嘛**取代**舊底料,要嘛**無效**。融合不在法術欄、在地板上。

> 為什麼是「取代不疊層」:**可讀性是 PvP 的硬天花板**(spec-elements §1)。
> 疊層 = N×N 組合爆炸,對手一眼讀不懂 → 砍。一格一狀態 = 永遠可秒讀。

---

## 1. 大原則:一格一狀態 + 招牌反應(取代,非疊層)

- **地板是一台狀態機**,每格只落在一個狀態(見 §3)。
- 元素落到某格 → 查「(現狀態 × 元素)」:
  1. **有招牌反應** → 轉成新的**單一**狀態(可能附一次性事件,如毒爆)。
  2. **可當底料** → **新底料取代舊底料**(油→水:水覆蓋)。
  3. **都不是** → **無效**(no-op)。
- **絕不出現「油+冰+毒」三層疊在同一格**。要嘛反應成一個新狀態,要嘛後者取代前者。

> 心智模型:元素是**轉場箭頭**,不是**圖層**。地板永遠只有「現在是什麼」一個答案。

## 2. 招牌反應表(~6 條,複用單機既有原語)

複用單機 `sim.js` 已驗證的反應原語(火燃油/草、雷導水、火引爆毒),只是搬到 PvP 地板語意:

| # | 觸發 | 現狀態 | 結果狀態 | 演出 / 效果 |
|---|---|---|---|---|
| R1 | 🔥 火 | 油 / 草 | **火海**(fire) | 沿**相連的油**擴散;站上去燒人、擋路 |
| R2 | ⚡ 雷 | 水 | **電水**(charged_water) | 踩進去 → 自電硬直(stun),經典自作自受 |
| R3 | 🔥 火 ↔ ☠️ 毒 | 毒(遇火) / 火(遇毒) | **清空**(clean) + 一次性毒爆 | 一次性 AoE 爆炸,炸完該格歸零 |
| R4 | ❄️ 冰 | 火 | **水**(water) | 熄滅 → 留下水(可再被 R2 電化) |
| R5 | ❄️ 冰 | 油 / 水 | **冰面**(ice) | 變滑 → 滑步/失控滑進艙/危險區 |
| R6 | 🌀 風 | (不改地板) | — | **不改地板**;吹動火/毒雲、把**人**送進上面任一狀態格 |

**未列的配對** = 走 §1 的規則 2/3(新底料取代 or 無效)。表只列**會產生新意義**的組合。

> 每條反應都通過那把尺(spec-elements §3):**改變空間 → 保留**;純扣血 → 降級。
> 毒的慢性扣血是**慌張計時器**(逼對手離開毒區),不是主要傷害來源。

## 3. 地板狀態機(格子狀態 + 轉場)

**狀態集**:`clean`(乾淨) / `oil`(油) / `water`(水) / `ice`(冰面) / `poison`(毒) / `fire`(火海) / `charged_water`(電水)。

轉場(箭頭 = 對該格丟某元素):

```
clean ──油──▶ oil ──🔥──▶ fire ──❄️──▶ water ──⚡──▶ charged_water
  │            │  ╲                        ▲              │
  │            │   ╲❄️                     │❄️(熄)        │(自然衰退)
  │            │    ▼                       │              ▼
  │            └──▶ ice ◀──────────────────┘            water
  │
  ├──水──▶ water ──❄️──▶ ice
  ├──毒──▶ poison ──🔥──▶ (毒爆) ──▶ clean
  └──(火/毒 相遇)──▶ (毒爆) ──▶ clean

fire  ──(相連 oil)──▶ 火海沿油擴散(R1)
風    ──▶ 不改格子狀態,只搬「人 / 雲」
所有狀態 ──(計時)──▶ 自然衰退回 clean(火海燒完、冰融、毒散)
```

- **底料互換**:`oil ⇄ water`(後丟者取代;水覆蓋油 = 撲滅可燃性,但不等於清潔)。
- **一次性事件**(毒爆)不是狀態,是「轉場時放一次的 AoE」,放完該格回 `clean`。
- **自然衰退**:每個危險狀態都有壽命 → 回 `clean`(避免地圖被永久污染鎖死)。

### 3.1 衰退壽命(起始值,之後實測微調)

**統馭原則:危險度 ↔ 壽命成反比。** 主動殺傷活越短(不能 camp);鋪陳/佈場活越久(才有設連段、逼位移的時間)。

| 組 | 狀態 | 起始壽命 | 理由 |
|---|---|---|---|
| 底料/鋪陳(長) | oil、water | **~10s** | 錨定「自 combo 跑動窗口」:油必須活得比「鋪→跑去撿火→回來」久一點點,否則自接永遠來不及 |
| 佈場/控制(中) | poison、ice | **~8s** | 逼位移要夠久才有壓力;毒 1 發活 8s 剛好值那 1 次 |
| 主動殺傷(短) | fire、charged_water | **~4s** | 強但短,火海不能當永久牆卡場;電荷比水本身短 |

兩個機制定案(決定衰退怎麼跑):

- **火海 = 滾動的浪**:每格火各自短命(~4s)+ 沿相連油傳播 → 火像波浪從點火處滾過油田、燒完即滅,是**會移動的危險**(好讀、戲劇、天然不 camp),**不是**整片同時燒的靜態牆。
- **電水衰退 → 乾淨**(不是變回水):雷等於**消耗**掉濕地——充一次電、放完電,該格歸零。濕地是一次性投資,不可反覆充電。
- **可讀性**:每個危險狀態**最後 ~1s 閃爍預警**再消失(純演出,實作順手加)。

## 4. 兩個已答情境(當實作驗收案例)

使用者提的兩個複合情境,結論即上表:

1. **「油地板燃燒中,另一人丟毒黏球到火上」**
   → `fire`(由 oil 點燃) 遇 **毒** = **R3 毒爆**:一次性 AoE 炸開、該格清空。
   *不是*「毒疊在火上」,是火毒相遇 → 一次性大事件 → 歸零。

2. **「地上是油,有人撿到冰霜瓶丟上去」**
   → `oil` 遇 **冰** = **R5 冰面**:油被**轉化成滑面**。
   關鍵:**不是單純取消/覆蓋**,而是把「可燃陷阱」變成「滑動陷阱」——
   性質改變(可燃 → 濕滑),仍是個能坑人的空間動詞。

> 這兩例正好示範兩種路徑:R3 = 招牌反應(新事件+歸零)、R5 = 招牌反應(性質轉化)。
> 都**不疊層**,都產生**一個新的單一狀態/事件**。

## 5. 已拍板的決策

1. **元素災難 = 工具與途徑**,把人**送進艙 / 打暈好抓**(混亂層)。
   與收容勝利**不衝突**:環境 = 擊殺 = 收容;元素災難是**餵給收容的前置**,不是平行傷害系統。
2. **觸發 = 自然定時 + 玩家觸發 混合**:場地會自己漏(定時災難源),玩家也能主動點燃/鋪設引爆。
3. **複合性靠地板、不靠法術欄**:玩家 A 鋪油 → 玩家 B 撿火棒點燃 → 玩家 C 撿毒球丟火上毒爆。
   多人各丟一手,地板把它們**串成一條因果**(有作者、有自作自受、有延遲引爆)。

## 6. 場地掛鉤(recycling center「洩漏事故」主題)

把地板化學接到已做好的工業回收場景(`render-lab.js`):

  冰面滑進、火海逼位、風壓直送,終點都是中央清運口。

## 7. 紀律(別越界)

- 這是 spec-elements 的**模型 ③ 變化層**:**現在設計、先不建**,等核心「3–4 人笑出聲」驗穩(規格 A)再上。
- 每條反應過尺:**改變空間 = 留;純 DPS = 砍**。反例:「火海傷害 +30%」。正例:「火點燃相連油成火海擋路」。
- **狀態數守恆**:狀態集控制在 ~7 個以內;要加新元素先問「會不會逼我加一排新狀態 → 破壞可讀性」。
- **不疊層是鐵則**:任何時候想「讓兩個效果同時在一格生效」,停——改成招牌反應(一個新狀態)或取代。

## 9. 元素 × 道具對照(建模藍圖)

核心邏輯:**七個元素 = 七種不重疊的投放原型**,動畫骨架才拉得開;命名維持「廢料魔法」調性。
每個道具的「連段角色」= 它在 §2 招牌反應裡的位置(道具 = 反應的載體)。

| 元素 | 定稿道具 | 投放原型 | 手感定位 | 魔法感重點 | 連段角色(→ §2) |
|---|---|---|---|---|---|
| **火** | 過熱核心噴火帽 | 噴流／衝撞 | 低頭前噴、近中距離火柱 | 藍白核心火、帽緣符文環、腳下法陣;火要「有秩序」 | 引燃油膜(R1) |
| **雷** | 魔導粗電線(電鞭) | 牽引 | 甩出纏住→灌電→拉近,最有記憶點 | 電流沿線流動符文、拉近時電弧 | 濕地導電加傷(R2) |
| **水** | 魔法水桶・壓 | 砸壓(由上而下領域) | 懸停蓄力→極快砸下,大開大合 | 流體**不潰散**、懸停停頓、發光濕痕 | 產出濕地接雷(R2) |
| **毒** | 紫色污染廢液桶 | 放置領域 | 倒地形成毒池/毒霧,拖延型 | 廢液發光漂浮、紫符文尾跡、毒霧浮光 | 火↔毒毒爆(R3)、控場逼位移 |
| **冰** | 冰霜瓶(地面冰刺) | 投擲 | 砸地後成範圍冰面 | 冰面竄出的霜白符文 | 冰面(R5),滑動逼位移 |
| **油** | 事故潤滑桶(油膜) | 投擲 | 潑油膜、地板變滑、可被引燃 | 油面反光符文、被點燃瞬間能量擴散 | **連段樞紐**(油+火、油+雷) |
| **風** | 風壓手套 | 近戰衝擊 | 出拳送氣浪、擊退、反彈投射物 | 氣流成形旋渦紋、拳套符文亮起 | 反彈、送人進艙(R6) |

**投放原型無重疊**:噴流(火)／牽引(雷)／砸壓(水)／放置(毒)／湧現(冰)／投擲(油)／近戰(風)——
七個各一種,骨架全不同,杜絕「換色而已」。**投擲只留油一個**(刻意的特例;油天生是連段引信)。

**貫穿原則**:元素要看起來**被意志控制、違反物理**(水該散不散、火該亂不亂、氣流成形)+ 發光符文與腳下法陣
→ 同時保住廢料喜感與魔法感,而非「意外在燒/在潑」。

### 9.1 四項拍板定案(2026-07 討論)

1. **土 = deferred(不刪,擱著)**。目前遊戲不出現;未來 netcode 成熟可能回來。
   砍的理由:造牆/裂地 = 唯一要**改幾何結構**的元素(每幀同步地形 + 重算碰撞),netcode 最貴;
   場地是固定回收中心,瓶頸已由場景設計死,主動造牆收益有限。油+水升為正式道具剛好補齊底料生產者,
   七件套自成閉環,土反而是唯一不進地板化學的孤兒。**送人進艙靠推/滑/送(風冰水),不靠擋(土)。**

2. **元素道具 vs 工具道具:概念分兩類、資料同一張表**(靠 `ITEM_SPEC.kind` 區分,見 item-cast 文件)。
   - **元素道具(這 7 個)**:走本表規則(原型不重疊、有連段角色、吃地板化學、元素色+符文+法陣)。
   - **工具道具**:不吃地板化學、自成一格、廢料科技風。**MVP 只放傳送一個**(`kind:'mobility'`、`whileDisabled:true`)。

3. **水 = 造濕地 + 砸中擊倒/暈一下(好抓)**。直擊傷害只是佐料,**擊殺理由是「濕地 → 接雷電水」**;
   砸中的**短擊倒**是空間動詞(剝奪移動、好抓送進艙),一魚兩吃(餵地板化學 **又** 服務「打暈好抓」)。
   刻意避免把「壓扁」做成一發打死的 DPS(Path-A 陷阱)。

4. **火/雷 MVP 先退化成單發**,塞現有單 impact 骨架;channel/多階段骨架驗穩後再擴(見 item-cast §「投放原型 × 骨架」)。
   - 火 MVP = 一發短噴(一個 impact 點燃前方油/造小塊火),**先不做持續火柱**。
   - 雷 MVP = 一發電擊(一個 impact,濕地加傷/自電),**先不做纏住拉近**。
   - 理由:火的遊戲價值在「點燃油成火海」(地板化學給的,不靠火柱持續);雷的記憶點「拉近」最貴,壓後面。

### 9.1.1 道具次數(3/2/1 三階)+ 自 combo

**次數 = 角色說明書**:數字本身教玩家這道具是哪種角色(撿到「油×3」直覺=拿來鋪的;「雷×1」=珍貴一擊)。
撿取時 `f.itemUses = ITEM_SPEC[type].uses`。

| 階 | 角色 | 次數 | 道具 | 理由 |
|---|---|---|---|---|
| A 鋪陳/推送 | 用量大、單發弱、靠反覆使用 | **3** | 油、風 | 油要潑夠多片才成連段;風是核心送人動詞要能反覆轟 |
| B 觸發/中堅 | 單發有份量、非終結 | **2** | 火、水 | 點燃/砸壓是關鍵一手,給兩發容錯 |
| C 終結/滯留/逃脫 | 強終結 / 一發夠久 / 保命 | **1** | 雷、冰、毒、傳送 | 雷=最強終結;冰毒=一發在地板留 8s(靠衰退撐時長);傳送=保命稀有 |

**1v1 自 combo = 允許,但有跑動成本**:一次只持有一個 `f.item`,自接「油→火」要「鋪油 → 跑去撿火 → 回來點」。
油壽命(~10s,見 §3.1)刻意設定成**撐得過這趟來回但很趕**,對手有機會離開油區 → 連段要移動+時間成本,不是一鍵。
(未來多人時自然變成多人各丟一手、地板串因果的原始願景。)

### 9.2 建模／落地順序(給建模用)

七個原型全模是大量工。表自身指出**油是樞紐**,故最小可玩迴圈只要三件套:

1. **第一批 · 油 + 火 + 風**(投擲鋪面 → 引燃成火海 → 把人轟進火海)。
   「鋪地板 → 點燃 → 送人進去」完整因果,能玩、能笑、能驗 fun。**先做這批。**
2. **第二批 · 水 + 雷**(造濕地 → 導電電水)。雷是最有記憶點但動畫+機制最貴,壓到第二批剛好。
3. **第三批 · 冰 + 毒**(控場、逼位移、補齊 R5/R3)。

每加一批 = 一組能玩的新連段,建模不用一次背七個。
