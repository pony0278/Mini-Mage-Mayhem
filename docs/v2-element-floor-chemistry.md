# v2 地板化學：元素反應系統（設計藍圖）

> 狀態：**設計定稿、尚未實作**（等核心「好笑」驗穩才動工——見 §7 紀律）。
> 定位：這是 [v2-spec-elements.md](v2-spec-elements.md) 說的**模型 ③「地圖撿元素」變化層**的地板端規則。
> 主軸不變：元素＝**空間動詞**（把人送進艙/打暈好抓的混亂層），不是 DPS。

---

## 0. 一句話

**一格地板只有一個狀態(取代,不疊層)。** 丟一個元素上去,要嘛觸發**招牌反應** →
變成**另一個單一狀態**,要嘛**取代**舊底料,要嘛**無效**。融合不在法術欄、在地板上。

> 為什麼是「取代不疊層」:**可讀性是 PvP 的硬天花板**(spec-elements §1)。
> 疊層 = N×N 組合爆炸,對手一眼讀不懂 → 砍。一格一狀態 = 永遠可秒讀。

---

## 1. 大原則:一格一狀態 + 招牌反應(取代,非疊層)

- **地板是一台狀態機**,每格只落在一個狀態(見 §3)。
- 元素落到某格 → 查「(現狀態 × 元素)」:
  1. **有招牌反應** → 轉成新的**單一**狀態(可能附一次性事件,如毒爆)。
  2. **可當底料** → **新底料取代舊底料**(油→水:水覆蓋)。
  3. **都不是** → **無效**(no-op)。
- **絕不出現「油+冰+毒」三層疊在同一格**。要嘛反應成一個新狀態,要嘛後者取代前者。

> 心智模型:元素是**轉場箭頭**,不是**圖層**。地板永遠只有「現在是什麼」一個答案。

---

## 2. 招牌反應表(~6 條,複用單機既有原語)

複用單機 `sim.js` 已驗證的反應原語(火燃油/草、雷導水、火引爆毒),只是搬到 PvP 地板語意:

| # | 觸發 | 現狀態 | 結果狀態 | 演出 / 效果 |
|---|---|---|---|---|
| R1 | 🔥 火 | 油 / 草 | **火海**(fire) | 沿**相連的油**擴散;站上去燒人、擋路 |
| R2 | ⚡ 雷 | 水 | **電水**(charged_water) | 踩進去 → 自電硬直(stun),經典自作自受 |
| R3 | 🔥 火 ↔ ☠️ 毒 | 毒(遇火) / 火(遇毒) | **清空**(clean) + 一次性毒爆 | 一次性 AoE 爆炸,炸完該格歸零 |
| R4 | ❄️ 冰 | 火 | **水**(water) | 熄滅 → 留下水(可再被 R2 電化) |
| R5 | ❄️ 冰 | 油 / 水 | **冰面**(ice) | 變滑 → 滑步/失控滑進艙/危險區 |
| R6 | 🌀 風 | (不改地板) | — | **不改地板**;吹動火/毒雲、把**人**送進上面任一狀態格 |

**未列的配對** = 走 §1 的規則 2/3(新底料取代 or 無效)。表只列**會產生新意義**的組合。

> 每條反應都通過那把尺(spec-elements §3):**改變空間 → 保留**;純扣血 → 降級。
> 毒的慢性扣血是**慌張計時器**(逼對手離開毒區),不是主要傷害來源。

## 3. 地板狀態機(格子狀態 + 轉場)

**狀態集**:`clean`(乾淨) / `oil`(油) / `water`(水) / `ice`(冰面) / `poison`(毒) / `fire`(火海) / `charged_water`(電水)。

轉場(箭頭 = 對該格丟某元素):

```
clean ──油──▶ oil ──🔥──▶ fire ──❄️──▶ water ──⚡──▶ charged_water
  │            │  ╲                        ▲              │
  │            │   ╲❄️                     │❄️(熄)        │(自然衰退)
  │            │    ▼                       │              ▼
  │            └──▶ ice ◀──────────────────┘            water
  │
  ├──水──▶ water ──❄️──▶ ice
  ├──毒──▶ poison ──🔥──▶ (毒爆) ──▶ clean
  └──(火/毒 相遇)──▶ (毒爆) ──▶ clean

fire  ──(相連 oil)──▶ 火海沿油擴散(R1)
風    ──▶ 不改格子狀態,只搬「人 / 雲」
所有狀態 ──(計時)──▶ 自然衰退回 clean(火海燒完、冰融、毒散)
```

- **底料互換**:`oil ⇄ water`(後丟者取代;水覆蓋油 = 撲滅可燃性,但不等於清潔)。
- **一次性事件**(毒爆)不是狀態,是「轉場時放一次的 AoE」,放完該格回 `clean`。
- **自然衰退**:每個危險狀態都有壽命 → 回 `clean`(避免地圖被永久污染鎖死)。

## 4. 兩個已答情境(當實作驗收案例)

使用者提的兩個複合情境,結論即上表:

1. **「油地板燃燒中,另一人丟毒黏球到火上」**
   → `fire`(由 oil 點燃) 遇 **毒** = **R3 毒爆**:一次性 AoE 炸開、該格清空。
   *不是*「毒疊在火上」,是火毒相遇 → 一次性大事件 → 歸零。

2. **「地上是油,有人撿到冰霜瓶丟上去」**
   → `oil` 遇 **冰** = **R5 冰面**:油被**轉化成滑面**。
   關鍵:**不是單純取消/覆蓋**,而是把「可燃陷阱」變成「滑動陷阱」——
   性質改變(可燃 → 濕滑),仍是個能坑人的空間動詞。

> 這兩例正好示範兩種路徑:R3 = 招牌反應(新事件+歸零)、R5 = 招牌反應(性質轉化)。
> 都**不疊層**,都產生**一個新的單一狀態/事件**。

## 5. 已拍板的決策

1. **元素災難 = 工具與途徑**,把人**送進艙 / 打暈好抓**(混亂層)。
   與收容勝利**不衝突**:環境 = 擊殺 = 收容;元素災難是**餵給收容的前置**,不是平行傷害系統。
2. **觸發 = 自然定時 + 玩家觸發 混合**:場地會自己漏(定時災難源),玩家也能主動點燃/鋪設引爆。
3. **複合性靠地板、不靠法術欄**:玩家 A 鋪油 → 玩家 B 撿火棒點燃 → 玩家 C 撿毒球丟火上毒爆。
   多人各丟一手,地板把它們**串成一條因果**(有作者、有自作自受、有延遲引爆)。

## 6. 場地掛鉤(recycling center「洩漏事故」主題)

把地板化學接到已做好的工業回收場景(`render-lab.js`):

- **主題**:回收中心的**元素洩漏事故** = 地板污染的世界觀理由。
- **四角元素站**(fire/frost/poison/lightning station)= 災難源:定時**洩漏 / 噴發**對應底料到鄰近地板。
- **四色分揀路線**(sorting routes)= 把災難**往中央輸送**的視覺引導(對齊「送進艙」)。
- **既有「失控入艙」**(擊飛/滑步/翻滾進 pod 半徑)= 地板動詞的**落地演出**——
  冰面滑進、火海逼位、風壓直送,終點都是中央清運口。

## 7. 紀律(別越界)

- 這是 spec-elements 的**模型 ③ 變化層**:**現在設計、先不建**,等核心「3–4 人笑出聲」驗穩(規格 A)再上。
- 每條反應過尺:**改變空間 = 留;純 DPS = 砍**。反例:「火海傷害 +30%」。正例:「火點燃相連油成火海擋路」。
- **狀態數守恆**:狀態集控制在 ~7 個以內;要加新元素先問「會不會逼我加一排新狀態 → 破壞可讀性」。
- **不疊層是鐵則**:任何時候想「讓兩個效果同時在一格生效」,停——改成招牌反應(一個新狀態)或取代。

## 8. 落地時的實作提示(給未來的自己)

- 資料:`game.map` 已是 `ROWS×COLS` 格 + `TILE_*` 常數(單機已有 water/ice/grass/burnt)。
  v2 地板狀態可**沿用/擴充同一格模型**,別另開平行結構。
- 反應查表:一張 `(state, element) → {next, event?}` 的 `FLOOR_RX` 表(鏡像 `ITEM_SPEC` 的正交表風格),
  加反應 = 加一列;查不到 = 走底料取代 / no-op。
- 衰退:每格帶一個 `ttl`,`update(dt)` 遞減歸零回 `clean`(比照單機 burnt/ice 的既有衰退)。
- 演出複用:火海/電水/冰面/毒區的粒子與地面標記,單機 `render-entities.js` 都已有原型。
