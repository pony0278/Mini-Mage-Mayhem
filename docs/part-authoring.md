# 部位建模規範(給建模師的一頁說明)

> 給 PUNCH STUDIO / Mini Mage Mayhem 做替換部位(GLB)時的建模約定。
> 一句話版本:**「檔案的原點就是接合點」——骨架不會去猜你的模型哪裡是手腕,它只會把
> 你檔案的 (0,0,0) 直接黏到關節上。**

## 為什麼要這樣做(插頭與插座)

把骨架想成一排**插座**(肩、肘、腕、髖、膝、踝…每個關節一個),你做的每個部位是一個**插頭**。
掛載的動作只有一件事:**把你檔案的原點 (0,0,0) 對到關節位置,不做任何額外旋轉或位移**。

所以系統「知道哪端是手腕」的方式,不是分析你的模型形狀——而是**約定**:

- 你把手腕切面放在原點 → 掛上去,手腕就剛好長在手臂末端 ✅
- 你的模型飄在離原點 0.2 的地方 → 掛上去,拳頭就飄在手腕旁邊 0.2 ❌(這不是 bug,是原點放錯)

方向同理:骨架假設你的部位**從原點沿 +Y 長出去、+Z 朝角色正面**。
方向錯了,掛上去就是整隻歪的。

## 三條通用規則(所有部位一體適用)

1. **原點 = 近端接縫的圓心**(「靠身體那端」的切面中心)
   手→手腕切面;前臂→手肘切面;上臂→肩膀切面;大腿→髖切面;小腿→膝切面;腳→腳踝切面;頭→頸切面。
2. **+Y = 從接縫往遠端長出去**
   手:腕→指節;前臂:肘→腕;大腿:髖→膝;頭:頸→頭頂。
3. **+Z = 角色正面**
   拳頭的指節面、腳尖、臉,都朝 +Z。

## 每個部位的規格表(單位=世界單位)

| 部位(命名) | 原點放哪 | +Y 指向 | 接縫半徑(±10%) | 標準長度 | 長度可自由? |
|---|---|---|---|---|---|
| `head` | 頸切面 | 頭頂 | 0.047 | 0.230 | ✅ 末端件,隨意(大頭free) |
| `neck` | 領口切面 | 頭方向 | 0.035 | 0.056 | ❌ 照表 |
| `upper_arm_l/r` | 肩切面 | 手肘方向 | 0.050 | 0.246 | ❌ 照表(或用 length_override) |
| `forearm_l/r` | 肘切面 | 手腕方向 | 0.022 | 0.205 | ❌ 照表 |
| `hand_l/r` | **腕切面** | **指節方向** | **0.037** | 0.099 | ✅ 末端件,**巨拳 free** |
| `thigh_l/r` | 髖切面 | 膝方向 | 0.030 | 0.233 | ❌ 照表 |
| `calf_l/r` | 膝切面 | 腳踝方向 | 0.025 | 0.209 | ❌ 照表 |
| `foot_l/r` | 踝切面 | 腳底方向 | 0.042 | 0.139 | ✅ 末端件,大鞋 free |
| `torso` | 脊柱基部 | 頭方向 | —(根件,無接縫) | — | — |

- 「接縫半徑」=切面圓環的半徑,誤差 ±10% 內可互換接合。
- 「末端件」(手/腳/頭)的長度純屬外觀,不牽動骨鏈——**浮誇比例免費**。
- 中段件(上臂/前臂/大小腿)的長度會影響下一個關節位置,要照標準長度。
- 左右件各做一份(`_l`/`_r`);鏡像由系統依骨骼實際邊位處理,建模時各自照同一套規則即可。

## Blender 匯出檢查清單

1. 部位擺好:**近端切面圓心=世界原點**、切面躺在 XZ 平面上、遠端朝 **+Y**、正面朝 **+Z**
2. `Ctrl+A` → **Apply All Transforms**(把位移/旋轉/縮放烘進網格——這步漏掉最常見)
3. 物件命名含部位關鍵字:`hand_l`、`left_hand`、`HAND_L` 都可以(大小寫、底線、連字號不拘)
4. 匯出 glTF/GLB,保持預設的 **+Y up**
5. 到 PUNCH STUDIO「＋ 逐一載入部位」→ 按「組裝檢視 T-pose」確認接合;
   微調用右側對位滑桿(x/y/z/rot/scale),調好「匯出對位 JSON」保存

## 常見症狀對照

| 症狀 | 原因 |
|---|---|
| 載入被拒(狀態列:無法判斷要掛到哪個 slot) | 檔名/節點名沒含部位關鍵字 |
| 部位飄在關節旁邊一段距離 | 原點沒放在接縫圓心(忘了移原點或 Apply Transforms) |
| 部位方向整隻歪/朝內 | +Y/+Z 軸向不對 |
| 接縫處粗細落差明顯 | 切面半徑偏離規格 >10% |
| 手臂被越推越長 | 中段件長度沒照表(誤把長度當外觀改) |
