## 後續階段（由近到遠）

### Phase 0 ｜ 手感快速打磨（小・先做・CP 值最高）
- **AI 難度**:現在偏強 → 加難度檔(調 `MASH_AI` / `STAB_*` / AI 推進積極度)。
- **攝影機位置**:現 flat `fov32/angle40/dist760` 跟隨 → 找更好角度。
- **地形微調**:平台大小 / 出生點 / 實驗艙位置。
- 驗收:每場立刻更舒服、AI 不再壓著打。

### Phase 1 ｜ 加深事故內容（中・核心投資）
- **法術/道具系統再定義**（見 [v2-spec-F-spells-items.md](v2-spec-F-spells-items.md)）:基礎動詞改「揮拳→擊暈→抓→搬→入倉」(陣風降為道具);場上撿即用道具取代升級樹。爆桶＝這層第一個原型。
- **環境危險接入**:水導電 / 油爆 / 冰滑 / 爆桶 —— 既是新削弱/混亂源,又解鎖新事故。
- **更多事故**:事故名稱 / 安全委員會吐槽 / 稱號 擴充(目標 ≥ 10 種圖鑑)。
- 這是「魔法事故報告」本體:報告夠雜夠好笑,分享才有意義。
- 驗收:每場報告都不太一樣、會想截圖。

### Phase 2 ｜ V1.0 社群分享（中・增長引擎）
- 分享卡圖片(1200×630)、挑戰碼(seed)、每日事故重現、最後 8 秒回放。
- **站在 Phase 1 的豐富報告上**才值得做。
- 驗收:有人真的把報告/挑戰碼貼出去。

### Phase 3 ｜ 美術主題:魔法實驗基地（中大・賣相）
- 白盒平台 → 房間化場地(實驗艙室/冷卻水池/導電地板/爆桶倉庫,見 spec E §6)。
- 複用浮島/體素美術。從「原型」變「作品」。
- 驗收:截圖看起來像一款遊戲,不是 debug 場。

### Phase 4 ｜ 玩法擴張（大・最後）
- 不對稱模式(失控法師 vs 收容員)、1vN、第二個動詞。
- **真多人 netcode**(最貴):等單機體驗夠厚、且確認值得才碰。
- 之後可接 spec E §7 的 V1.1+(基地/地圖編輯器、非同步 PvP)。
