# v2 規格 F — 法術與道具系統（撿即用 / 事故來源）

> 來源願景：使用者設計稿《魔法事故報告：法術與道具系統設計稿》→ [magic-spell-item-concept.md](magic-spell-item-concept.md)（完整原文）。

> 本檔＝**可執行 spec**：把一輪討論的定案釘成「基礎動詞 + 數值 / 道具規則 + 操作 / MVP 道具清單 + 各道具效果 / 收容判定 / 報告欄位」，並對照現有 codebase 的可重用零件。設計問題已全部拍板，剩下的只有實測後的數字微調（見 §5 末）。

## 0. 一句話再定位

不是「農經驗升級湊 build」，而是 **「在失控的魔法實驗基地裡，撿散落的危險魔法道具，邊追捕邊亂用，把整場搞成一份事故報告」**。
法術＝**事故來源**，不是穩定武器。護城河仍是規格 E：**輸了也好笑、好笑到想截圖分享**。

> 北極星不變：玩完一局會說「這局也太白癡了，我要給朋友看」。

## 1. 為什麼不主打「吃經驗升級 3 選 1」（概念稿 §2/§3）

升級樹會把注意力導向「農經驗、湊 build、升級傷害、合成大招」→ 逐漸退回普通 Roguelike Arena。
我們真正有辨識度的是「把失控玩家抓進收容艙，中途各種道具/機關失控，最後生成好笑事故報告」。
所以主系統服務於**追捕 / 逃跑 / 搶道具 / 抓人 / 掙脫 / 誤傷 / 反向收容 / 場地事故 / 分享性結算**，而非刷怪升級。

系統比例（概念稿 §1）：**場上道具 70% ／ 角色初始傾向 20% ／ 事故異變 10%**（異變 MVP 先砍，見 §6）。

## 2. 基礎動詞（免費、人人都會）— ✅ 定案

把規格 E 的「遠程陣風推進艙」換成 **近戰抓捕**。

> 為什麼換：遠程陣風＝整場互相彈開，**吵、滑、淺**，每次交手沒有結構。近戰「打崩 → 抓走」有節奏，主題上就是收容員，反轉還天然內建。

### 2.1 狀態機

```
中立(互相揮拳試探)
  → 揮拳(近距攻擊弧, 命中削穩定值 + 小硬直)
  → 穩定值歸零 = 擊暈(1~1.5s 不能動, 頭上冒星 = 可被抓窗口)
  → 靠近按鍵「抓起」(扛/拖著, 你移動變慢)
  → 搬運(對方狂按掙脫; 填滿掙脫值 = 掙開 + 短暫反擊窗)
  → 拖進收容艙半徑 = 收容成功(該回合勝)
```

- **掙脫在「搬運途中」全程進行**（新機制的緊張感與反轉都在這段）。複用規格 E 的 `escape` mash 系統，只是從「艙邊倒數」搬到「搬運中」。
- **反轉天然長出**：搬運時你變慢變脆弱 → 被掙脫 / 被打斷 → 換你被抓走。整場遊戲要的「我快抓到他，結果自己被關」從基礎機制自己冒出來。
- **不再整場滑行**：滑行從「基礎噪音」變成「道具高光」（見 §3）。

### 2.2 收容判定：失控入艙 — ✅ 定案

> **在「失控狀態」下進入實驗艙半徑 → 收容成立；在「正常操控」下踏進艙 → 沒事。**
> 失控狀態 ＝ 被搬運中 / 暈眩中 / 被擊退滑行中 / 打滑中。

一條規則統一所有收容路徑：親手搬入（被搬＝失控）、被風吹入、被冰滑入都算收容；正常走位踏過艙不會誤收容自己。收容從單一路徑變成「**把對手弄成失控、再讓他進艙**」的開放解（搬 / 吹 / 滑 / 炸飛皆合法）——正是「散落道具製造收容事故」的精神。遠程手段（風 / 冰）能收容但難瞄，偏事故而非穩定戰術。

### 2.3 抓 / 搬 / 掙脫 / 入倉 數值 — ✅ 定案（起始值，實測後調）

> 設計洞見：**搬運速度 × 掙脫時間 ＝ 有效抓捕半徑**。搬運 ~104px/s × 掙脫 ~2.2s ≈ 只能拖 ~230px → 在艙附近打贏才抓得成、角落抓不到 → 戰鬥自然被拉向實驗艙，道具（風 / 傳送）成為「遠處也能完成抓捕」的關鍵。

| 階段 | 起始值 |
|---|---|
| 揮拳 | 射程 `46`、錐角 `±0.9rad`、CD `0.35s`、削穩定值 `25`（≈4 下擊暈）、輕擊退 `~130` + hitstop、命中暫停對方回穩 `0.8s` |
| 擊暈 | 穩定值歸零 → 暈眩 `1.2s`（不能動、冒★）；沒被抓 → 回穩到 `40` + `0.6s` 免疫再暈 |
| 抓 | 對暈眩者距離 ≤`46` 按動作鍵 → 搬運；搬運中移速 `×0.6`（~104px/s）、不能揮拳 / 用道具 |
| 搬運＋掙脫 | 全程可掙脫；**按指示左右交替點按**（同鍵猛敲不算），交替順暢 `~2.2s` 掙開；AI 固定填速（等效 ~2.2s）。傳送符可瞬間逃脫（見 §4.2）|
| 掙脫成功 | 搬運者踉蹌 `0.5s`（可被打）、被搬者以 `50` 穩定值脫身 → **反轉窗口** |
| 入艙 | 拖進艙半徑 ＝ **瞬間收容成功**（該回合勝）；只加 `0.3s` 封艙演出，不再倒數 |

### 2.4 初始技能量（概念稿 §5）

每位玩家進場最多：**1 個基礎動作（拳+抓）＋ 1 個通用動作（閃避/短衝）**。不要一開始給太多技能。

## 3. 道具層規則 — ✅ 定案

- **陣風降為道具**（風壓手套）。想把人吹飛 / 吹進艙 / 吹下海時才拿得到 → 滑行變成偶爾的爆點。
- **進場：固定補給座 + 定時重刷**（複用爆桶的「定點 + 重生」邏輯，動線可預期、最省工）。
- **庫存：只能拿 1 個**；手上有道具時**補給座對你無效，必須先用掉才能再撿** → **強迫使用**，等於系統主動生產「為了空手搶下一個就先亂用」的行為 = 事故素材。

### 3.1 設計鐵則（每個道具都要過這三關）

1. **落在 5 動詞之一**：位移 / 控制 / 穩定值 / 艙門狀態 / 場地。
2. **真的影響收容過程**（抓 / 搬 / 掙脫 / 入倉 / 艙門）——答不出「它怎麼影響收容艙」就不做。
3. **在 1v1 就會笑**（自反噬 / 反轉——因為沒隊友可誤傷，1v1 喜劇靠自作自受與反轉）。

> 註：**「穩定值削弱」已由基礎揮拳承包**，道具不需再做削弱型；**「艙門短路 / 爆炸」已由爆桶承包**。道具主要補**位移 / 控制 / 場地 / 艙門干擾**這幾格。

### 3.2 道具 → 收容的鉤子（對照現有 `lock` 狀態機）

現有：`lock = { pid, escape, ... }`（搬運掙脫）＋ `POD`（艙）＋ `podOverload()`（過載反轉，爆桶已在用）。道具就是去改這些：

| 互動 | 掛在哪 |
|---|---|
| 助攻把暈眩者吹向艙 | 風把目標朝 `POD` 推 |
| 打斷搬運 / 鬧鬆手 | 對搬運者施力 → 強制鬆手 |
| 反向 / 短路 | 呼叫 `podOverload()`（同爆桶炸到艙）|
| 逃脫 / 反轉 | 傳送換位 |
| 破壞搬運（滑） | 冰面 → 搬運者打滑鬆手 / 被丟者滑進艙 |

### 3.3 操作與按鍵 — ✅ 定案

- **揮拳與抓合成一顆情境「動作鍵」**：附近有暈眩對手 → 抓；否則 → 揮拳；搬運中 → 放下。
- 建議配置：**WASD 移動｜J（或滑鼠左鍵）動作｜K（或滑鼠右鍵）用道具**。面向＝移動方向（沿用現有；滑鼠瞄準之後可加）。只要 **2 顆動作鍵** → 對手機觸控友善。
- **持道具與抓人各自獨立（甲案）**：持道具**不**擋抓人；但**搬運中兩手全滿 → 不能用道具 / 揮拳，只能移動或放下**。（「兩手原則」＝持道具就不能抓，留作之後想加深時的旋鈕。）

## 4. MVP 道具集 — ✅ 定案：3 撿取道具 + 現有爆桶

> **風壓手套（遠程位移）＋ 傳送符（反轉）＋ 冰霜瓶（場地/滑）＋ 爆桶（環境·艙門短路，已上線）**

| 道具 | 動詞 | 收容鉤子 | 1v1 笑點 | 工 |
|---|---|---|---|---|
| **風壓手套** | 位移(遠程) | 吹暈眩者向艙助攻 / 吹搬運者鬆手 / 把人吹進艙 | 風壓過載把自己往後彈 | 極低（複用現有陣風）|
| **傳送符** | 位移(換位/反轉) | 被搬時換位＝瞬間逃脫；對調快到艙口的局面 | 招牌反轉：我快抓到他 → 換位 → 換我被抓 | 低 |
| **冰霜瓶** | 場地/控制(滑) | 艙口結冰 → 搬運者打滑鬆手 / 被丟者滑進艙 | 自己丟的冰自己踩滑；滑進艙（自作自受梗搬家到這）| 中 |
| **爆桶**（已上線）| 艙門(短路) + 位移(炸飛) | 炸到艙 → 過載反轉 | 炸飛自己 | 已做 |

**動詞覆蓋乾淨不重複**：拳=削弱 ／ 風=遠程位移 ／ 傳送=反轉 ／ 冰=場地控制 ／ 爆桶=艙門短路。五格全中，沒有兩個道具做同一件事。

### 4.1 🌀 風壓手套（位移·遠程）— 危險度：注意級
- **效果**：朝面向發射風錐，命中沿面向強力擊退（稀有道具，比舊免費陣風更有份量）。起始值：射程 `150`、錐角 `±1.0rad`、力道推 ~6 格（`~190px`，WEIGHTY 摩擦乾脆停）；命中期間對手＝被擊退滑行中（失控）。
- **艙門鉤子**：①吹對手在擊退失控中滑進艙半徑 → 遠程收容（難瞄＝事故味）②吹中搬運者 → 強制鬆手打斷抓捕 ③吹開逼近艙口者（防守）。
- **自反噬**：風壓過載——貼臉（<`50px`）發射反作用力把自己往後彈（可能自彈進艙 / 爆桶）。

### 4.2 🌀 傳送符（位移·反轉）— 危險度：危險級
- **效果（自由時）**：與面向最近對手**換位**；範圍內無人 → 往面向短距瞬移 `~150px`。
- **被抓時**：瞬間換位＋解抓＋搬運者踉蹌 `0.5s`——**唯一可在被搬運中使用的道具（＝逃脫身份）**。
- **艙門鉤子**：①被搬向艙時換位脫身 ②把艙口附近對手與自己對調 ③**兩步反殺**：換位 → 對手踉蹌 → 反手打暈就近拖入艙（「收容員自己被關」）。
- **自反噬**：傳送不穩定——落點 `±20px` 偏移，偶爾送自己到爆桶旁 / 艙口自己那側。

### 4.3 ❄️ 冰霜瓶（場地/控制·滑）— 危險度：注意級
- **效果**：朝面向丟出，落地生成冰面（半徑 `~60`、持續 `~5s`）。踩上＝打滑中（失控；低摩擦、難急停、會滑）。
- **艙門鉤子**：①艙口鋪冰 → 搬運者打滑鬆手、難精準拖入 ②被推 / 被搬經過冰面者打滑滑進艙 → 收容（滑進艙笑點）③封路（追兵打滑）。
- **自反噬**：自己丟的冰自己踩滑；追人滑過頭撞牆 / 滑進爆桶範圍。

**延後（有理由）**：火球卷軸 / 雷擊棒（「爆炸擊飛 + 炸艙門短路」爆桶已佔）、泡泡槍（拳本來就能開抓窗口）、超載核心 / 磁力石 / 黏液桶 / 反重力球（禁用級 / 需支援內容如金屬物件 / 多人才發揮）。

## 5. 定案總表（原待定 #1~#5 已拍板）

1. **抓/搬/掙脫/入倉數值**：採 §2.3 起始值，**實測後再調**；入艙＝瞬間成功（§2.3）。
2. **操作**：揮拳+抓合成情境動作鍵；`WASD｜J 動作｜K 道具`；**持道具與抓人各自獨立（甲）**、搬運中兩手全滿（§3.3）。
3. **傳送符 ＝ 被搬運中唯一可用的道具**（逃脫身份）；效果＝**換位**（§4.2）。一般掙脫＝按指示左右交替；傳送＝瞬間逃脫。
4. **3 道具精確效果 / 艙門鉤子**：見 §4.1~4.3；**報告欄位**見 §7。
5. **角色初始傾向**：MVP **對稱、不做傾向**。傾向留作之後的「角色選擇層」，以「**基礎修飾＋收容風格**」形式（風=推位／土=護艙／冰=控場／雷=打斷／火=壓制），**不做獨立法術樹**；與「不對稱陣營（失控法師 vs 收容員）」是**另一條**之後的軸。

> **真正留待實測微調**（是手感數字，不是設計問題）：掙脫難度（§2.3 的 ~2.2s）、入艙要不要補艙口短倒數、風力道、冰面持續、傳送範圍上限。

## 6. 砍掉什麼 / 不刪什麼

- **MVP 砍「事故能量 + 3 選 1 異變」**（概念稿 §9/§10）。原因：短局（先贏 3 回合）易重建雪球（單方累積、中途躍升、落後方更難翻）；且道具層已提供足夠變化，異變功能重複。
  → **暫存**；日後要隨機味，走 **「每回合對稱基地故障」**（雙方共享的環境異常，例：這回合地板偏滑 / 爆桶更多 / 艙門易短路），**不走個人累積雪球**。
- **法術一個都不刪**：
  - 單機 `sim.js` 的完整元素 / 融合 / 反應 / 畢業大絕 **照常上線**，同時是**零件捐贈庫**。
  - 概念稿的道具表（§7）/ 五系終極（§12）/ 元素反應（§13）＝**待重新包裝的素材**：火球卷軸 = 火球法術包成一次性道具；§13 元素反應 = 直接沿用單機反應引擎，**輸出從「傷害」改成「事故」**（誤傷 / 滑倒 / 艙門異常 / 位移）。

> 一路原則不變：**沒有東西被浪費**——先前的元素 / 融合 / 反應 / 大絕全是餵這套的零件。

## 7. 餵事故報告（概念稿 §16）— ✅ 定案欄位

沿用規格 E 的報告 schema / 等級 / 委員會吐槽，`inc.*` 擴以下欄位：

| 計數器 | 記什麼 | 報告用途 |
|---|---|---|
| `itemUses{wind,teleport,ice}` | 各道具使用次數 | 主要涉案道具 |
| `carries` | 親手搬入收容 | 收容方式＝搬 |
| `accidentContains{wind,ice}` | 吹入 / 滑入的失控入艙收容 | 收容方式＝吹 / 滑 |
| `reverseContains` | 反向收容（換位反殺 / 收容員被關）| 招牌反轉 |
| `teleportEscapes` | 傳送脫逃 | 空間錯亂 |
| `struggleEscapes` | 狂按掙脫成功 | 逃脫戲 |
| `itemBackfires` | 被自己道具害到 | 自作自受 |
| `overloads` / `barrelBooms` | 艙門短路 / 爆桶（沿用）| — |

**荒謬度 → 等級**（沿用現有 `chaos → level`，換權重）：

```
chaos = carries×1 + accidentContains×2 + overloads×2
      + reverseContains×3 + itemBackfires×1 + barrelBooms×1
      + 涉案「危險級」道具種類數×1        ← 概念稿 §8
```

**報告新增欄位**：主要涉案道具、收容方式（搬 / 吹 / 滑）、反向收容 N、自傷 N、艙門短路 N。

**道具驅動的新事故名稱 / 稱號 / 評語**（接現有查表）：
- `reverseContains≥1` → 「反向收容事件」/ 稱號「換位藝術家」/ 評語「技術上來說，有人被成功收容了。只是收錯人。」
- `itemBackfires≥2` → 「自體事故頻發」/「自助受測體」
- `accidentContains≥1` → 「自投羅網事件」
- `itemUses.teleport≥3` → 「空間錯亂事件」
- 概念 §8：涉案危險級道具越多 → 「本次事故涉及 N 件危險級道具，事故等級提升。」
