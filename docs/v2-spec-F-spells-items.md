## 0. 一句話再定位

不是「農經驗升級湊 build」，而是 **「在失控的魔法實驗基地裡，撿散落的危險魔法道具，邊追捕邊亂用，把整場搞成一份事故報告」**。
法術＝**事故來源**，不是穩定武器。護城河仍是規格 E：**輸了也好笑、好笑到想截圖分享**。

> 北極星不變：玩完一局會說「這局也太白癡了，我要給朋友看」。

---

## 1. 為什麼不主打「吃經驗升級 3 選 1」（概念稿 §2/§3）

升級樹會把注意力導向「農經驗、湊 build、升級傷害、合成大招」→ 逐漸退回普通 Roguelike Arena。
我們真正有辨識度的是「把失控玩家抓進收容艙，中途各種道具/機關失控，最後生成好笑事故報告」。
所以主系統服務於**追捕 / 逃跑 / 搶道具 / 抓人 / 掙脫 / 誤傷 / 反向收容 / 場地事故 / 分享性結算**，而非刷怪升級。

系統比例（概念稿 §1）：**場上道具 70% ／ 角色初始傾向 20% ／ 事故異變 10%**（異變 MVP 先砍，見 §6）。

## 2. 基礎動詞（免費、人人都會）— ✅ 定案

把規格 E 的「遠程陣風推進艙」換成 **近戰抓捕**。

> 為什麼換：遠程陣風＝整場互相彈開，**吵、滑、淺**，每次交手沒有結構。近戰「打崩 → 抓走」有節奏，主題上就是收容員，反轉還天然內建。

### 2.1 狀態機

```
中立(互相揮拳試探)
  → 揮拳(近距攻擊弧, 命中削穩定值 + 小硬直)
  → 穩定值歸零 = 擊暈(1~1.5s 不能動, 頭上冒星 = 可被抓窗口)
  → 靠近按鍵「抓起」(扛/拖著, 你移動變慢)
  → 搬運(對方狂按掙脫; 填滿掙脫值 = 掙開 + 短暫反擊窗)
  → 拖進收容艙半徑 = 收容成功(該回合勝)
```

- **掙脫在「搬運途中」全程進行**（新機制的緊張感與反轉都在這段）。複用規格 E 的 `escape` mash 系統，只是從「艙邊倒數」搬到「搬運中」。
- **反轉天然長出**：搬運時你變慢變脆弱 → 被掙脫 / 被打斷 → 換你被抓走。整場遊戲要的「我快抓到他，結果自己被關」從基礎機制自己冒出來。
- **不再整場滑行**：滑行從「基礎噪音」變成「道具高光」（見 §3）。

### 2.2 初始技能量（概念稿 §5）

每位玩家進場最多：**1 個基礎動作（拳+抓）＋ 1 個通用動作（閃避/短衝）**。不要一開始給太多技能。

## 3. 道具層規則 — ✅ 定案

- **陣風降為道具**（風壓手套）。想把人吹飛 / 吹進艙 / 吹下海時才拿得到 → 滑行變成偶爾的爆點。
- **進場：固定補給座 + 定時重刷**（複用爆桶的「定點 + 重生」邏輯，動線可預期、最省工）。
- **庫存：只能拿 1 個**；手上有道具時**補給座對你無效，必須先用掉才能再撿** → **強迫使用**，等於系統主動生產「為了空手搶下一個就先亂用」的行為 = 事故素材。

### 3.1 設計鐵則（每個道具都要過這三關）

1. **落在 5 動詞之一**：位移 / 控制 / 穩定值 / 艙門狀態 / 場地。
2. **真的影響收容過程**（抓 / 搬 / 掙脫 / 入倉 / 艙門）——答不出「它怎麼影響收容艙」就不做。
3. **在 1v1 就會笑**（自反噬 / 反轉——因為沒隊友可誤傷，1v1 喜劇靠自作自受與反轉）。

> 註：**「穩定值削弱」已由基礎揮拳承包**，道具不需再做削弱型；**「艙門短路 / 爆炸」已由爆桶承包**。道具主要補**位移 / 控制 / 場地 / 艙門干擾**這幾格。

### 3.2 道具 → 收容的鉤子（對照現有 `lock` 狀態機）

現有：`lock = { pid, escape, ... }`（搬運掙脫）＋ `POD`（艙）＋ `podOverload()`（過載反轉，爆桶已在用）。道具就是去改這些：

| 互動 | 掛在哪 |
|---|---|
| 助攻把暈眩者吹向艙 | 風把目標朝 `POD` 推 |
| 打斷搬運 / 鬧鬆手 | 對搬運者施力 → 強制鬆手 |
| 反向 / 短路 | 呼叫 `podOverload()`（同爆桶炸到艙）|
| 逃脫 / 反轉 | 傳送換位 |
| 破壞搬運（滑） | 冰面 → 搬運者打滑鬆手 / 被丟者滑進艙 |

## 4. MVP 道具集 — ✅ 定案：3 撿取道具 + 現有爆桶

> **風壓手套（遠程位移）＋ 傳送符（反轉）＋ 冰霜瓶（場地/滑）＋ 爆桶（環境·艙門短路，已上線）**

| 道具 | 動詞 | 收容鉤子 | 1v1 笑點 | 工 |
|---|---|---|---|---|
| **風壓手套** | 位移(遠程) | 吹暈眩者向艙助攻 / 吹搬運者鬆手 / 把人吹進艙 | 風壓過載把自己往後彈 | 極低（複用現有陣風）|
| **傳送符** | 位移(換位/反轉) | 被搬時換位＝瞬間逃脫；對調快到艙口的局面 | 招牌反轉：我快抓到他 → 換位 → 換我被抓 | 低 |
| **冰霜瓶** | 場地/控制(滑) | 艙口結冰 → 搬運者打滑鬆手 / 被丟者滑進艙 | 自己丟的冰自己踩滑；滑進艙（自作自受梗搬家到這）| 中 |
| **爆桶**（已上線）| 艙門(短路) + 位移(炸飛) | 炸到艙 → 過載反轉 | 炸飛自己 | 已做 |

**動詞覆蓋乾淨不重複**：拳=削弱 ／ 風=遠程位移 ／ 傳送=反轉 ／ 冰=場地控制 ／ 爆桶=艙門短路。五格全中，沒有兩個道具做同一件事。

**延後（有理由）**：火球卷軸 / 雷擊棒（「爆炸擊飛 + 炸艙門短路」爆桶已佔）、泡泡槍（拳本來就能開抓窗口）、超載核心 / 磁力石 / 黏液桶 / 反重力球（禁用級 / 需支援內容如金屬物件 / 多人才發揮）。

## 5. 待定清單（進原型再拍）

1. **抓 / 搬 / 掙脫 / 入倉的細節數值**：掙脫填速、鬆手條件、拖進艙是瞬間成功還是艙口再一小段倒數。
2. **按鍵配置** + 「拿著道具時能不能同時抓人搬運」。
3. **傳送符是否為「被抓搬運中唯一可用的道具」**（＝逃脫身份，其他道具被抓時手被壓住用不了）——傾向：是。
4. **各道具精確數值 / 艙門鉤子** + **報告新增哪些道具統計欄位**（見 §7）。
5. **角色初始「法術傾向」**（概念稿 §5 的 20%）：MVP **傾向先對稱、不做傾向**（雙方一樣，只靠撿到的道具區分），傾向系統當之後的「角色選擇層」。

## 6. 砍掉什麼 / 不刪什麼

- **MVP 砍「事故能量 + 3 選 1 異變」**（概念稿 §9/§10）。原因：短局（先贏 3 回合）易重建雪球（單方累積、中途躍升、落後方更難翻）；且道具層已提供足夠變化，異變功能重複。
  → **暫存**；日後要隨機味，走 **「每回合對稱基地故障」**（雙方共享的環境異常，例：這回合地板偏滑 / 爆桶更多 / 艙門易短路），**不走個人累積雪球**。
- **法術一個都不刪**：
  - 單機 `sim.js` 的完整元素 / 融合 / 反應 / 畢業大絕 **照常上線**，同時是**零件捐贈庫**。
  - 概念稿的道具表（§7）/ 五系終極（§12）/ 元素反應（§13）＝**待重新包裝的素材**：火球卷軸 = 火球法術包成一次性道具；§13 元素反應 = 直接沿用單機反應引擎，**輸出從「傷害」改成「事故」**（誤傷 / 滑倒 / 艙門異常 / 位移）。

> 一路原則不變：**沒有東西被浪費**——先前的元素 / 融合 / 反應 / 大絕全是餵這套的零件。

## 7. 餵事故報告（概念稿 §16）

結算報告自動讀本局關鍵事件：最常用道具 / 造成最大事故的道具 / 最後觸發的元素反應 / 是否反向收容 / 是否自傷 / 是否艙門短路 / 基地損害% …。
現有 `inc.*` 計數器 + `generateReport()` **擴欄位**即可（沿用規格 E 的報告 schema / 等級 / 委員會吐槽）。

## 9. 一句話

不是：

> 我升級出了最強法術。

而是：

> **我撿到一個危險卷軸，結果把對手、自己和收容艙一起搞壞了。**
