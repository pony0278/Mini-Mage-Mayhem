# 魔法事故回收中心:核心玩法憲章 v1.1

## 0. 版本沿革(為什麼改)

| # | v1 → v1.1 的變更 | 原因 |
|---|---|---|
| 1 | 勝負來源:收容 3 次獲勝 → **先完成 3 組分類序列=提前下班**(工作進度是唯一勝利) | v1 §3.1 鎖死規則;避免退化成普通格鬥遊戲 |
| 2 | 怒氣→老闆暴走→轉場純戰鬥 **整條線退役**;老闆降為開場/結算的框架角色 | v1 §5.2「只保留一條可見能量」;戰鬥改為全程可用的干擾,不需要模式切換 |
| 3 | AI 怒氣表 → **雙方各一條「事故能量」**(v2s.anger 一般化) | 同上;被打方充能=自動防雪球(v1 §3.3) |
| 4 | 第一技能從「風壓推擊」改為 **能量閘控現有擊暈鏈**(拳→暈→抓→丟艙) | 對現有代碼最小改動;擊暈鏈本身就是最有戲的技能 |
| 5 | 高潮定案**讀法 B**:丟進回收口=約 3 秒停工(不直接獲勝),你趁機分完最後一件→下班鐘響→贏;若對手此刻仍在艙內,結算演出直接封艙 | v1 §3.5/§9.4;中途干擾與結局懲罰在演出上無縫接軌 |
| 6 | 充能來源定案:**分對也充能**(工作為主要來源、被打是加速器) | 若只有被打充能,領先且沒被欺負的人永遠解不開能力;工作充能讓終盤雙方自然都武裝=高潮期 |

## 1. 一句話玩法

> 玩家與 AI 按照公開的元素序列搶著分類垃圾,率先完成工作的人提前下班;分類與受擊都會累積事故能量,能量達標才能擊暈對手製造時間差——但只顧打架的人,對手會趁亂完成最後一件工作。

任何未來新增的系統,都必須回答:**它是否讓「分類競速、戰術干擾、提前下班」變得更有趣?** 答案不明確就暫時不加入。

## 2. 核心循環(定案六步)

```text
① 開場:老闆在 UI 出現,監督玩家與 AI「好好工作」→ 開始後老闆退場(結算時再回來)
② 場上散落元素垃圾;公開元素序列 UI 指示當前要撿什麼 → 投入中央回收艙
③ 部分元素刻意稀缺(場上只有一件)→ 攻擊 AI 搶資源(打=踉蹌+掉瓶,不會擊暈)
④ 元素瓶會破裂;丟歪砸到 AI = AI 生氣 → 「被砸那方」累積事故能量(欺負 AI=武裝 AI)
⑤ 事故能量未達定值,無法擊暈對方(擊暈鏈=能量技能)
⑥ 搶第一個下班 → 終盤衝突 → 擊暈對方丟入回收艙(≈3 秒停工)→ 趁機分完最後一件 → 下班!
   (對手若仍在艙內 → 結算演出直接封艙壓縮清運=輸家的下班懲罰)
```

## 3. 五個不可動搖的設計支柱(繼承 v1,全文照舊)

1. **工作才是唯一勝利來源**:正確分類→推進下班進度→率先完成→提前下班。攻擊/推人/打暈/回收對手都不能直接加勝利分數。
2. **干擾只能製造時間差**:掉瓶、短暫停工、搶路線、逼改計畫。核心選擇=「繼續工作,還是花時間拖慢對方?」
3. **被攻擊者必須得到反擊資源**:被正式命中→事故能量快速增加。防雪球細則:同段連擊只計一次、受擊後短時間不再獲得、普通碰撞不計、被連續壓制給短暫控制免疫。
4. **混亂必須可理解、可恢復**:普通推開 0.3~0.8s、掉物立即可動、冰凍/暈眩 1.5~2.5s、中央回收最多約 3s、受擊保護 1~1.5s。要「可惡,差一點!」不要「我什麼都不能做」。
5. **中央回收口是招牌演出,不是主要規則**:能量滿→製造破綻→丟入→對手短暫停工→從其他出口被吐回。不直接決定勝負——**唯一例外**:下班鐘響時對手剛好在艙內,演出順勢封艙(讀法 B 的結局合體)。

## 4. 兩條資源

### 4.1 下班進度(唯一勝利進度)
```text
完成一組序列 → 下班進度 +1;率先 3 格 → 提前下班
玩家:■□□   AI:■□□   (雙方進度並排顯示)
```

### 4.2 事故能量(唯一能力資源;雙方各一條)
| 行為 | 能量 |
|---|---:|
| 正確投入一件垃圾 | +10%~15% |
| 完成一組序列 | 額外 +10% |
| 被正式攻擊/瓶砸命中 | +20%~30%(充**被打方**) |
| 普通推擠/碰撞 | 0 |
| 分類錯誤 | 0 |
| 攻擊命中別人 | 0(打人不養自己的條) |

能量達標解鎖:**擊暈能力**(拳的第三段才可暈/或蓄力拳,實作再定)→ 暈→抓→丟艙。未達標的拳永遠只是踉蹌+掉瓶。
視覺:工作累積=紫色魔法能量、受擊累積=轉紅冒煙、能量滿=角色進入事故準備狀態(AI 滿=冒煙,v1 §10.4 的怒氣載體)。

## 5. 序列與供料

- 公開序列一組 4 個元素(例 🔥→🔥→❄️→⚡),**雙方看同一組謎題、進度各自獨立**(一方完成不替另一方推進)。
- 當前需求持續放大/發光;完成打勾前進。序列常駐顯示,不要求背誦。
- **供料保證+稀缺窗**:respawn 換型加權偏向「雙方當前步驟需要的元素」,但保留數秒稀缺窗——窗內=賽跑/互搶的戲。窗長可調參數。
- 垃圾四型沿用現況 fire/ice/poison/lightning(§13 待決:是否把毒改水對齊 v1 圖例)。

## 6. 干擾規則(定案)

- **免費層(隨時)**:拳=踉蹌 0.3~0.8s+打掉手上瓶,**不暈**。丟瓶砸人=同級干擾+瓶碎(=攻擊自帶資源損失的機會成本)。
- **能量層(達標後)**:擊暈(1.5~2.5s)→ 可抓→可丟艙(≈3s 停工,從其他出口吐回)。
- **平衡鐵則**(v1 §8):普通干擾造成的對手時間損失 ≤ 使用者自身付出時間的 2 倍;只有滿能量的回收技能可到 3 秒級。
- 分錯=輕罰:設備拒收、瓶彈回(不銷毀)、近距投入者停頓約 0.4s、序列不前進。**不觸發事故噴發**(v0.8 的 eruptStation 分錯事故退役,保留代碼待後續當升級事件)。

## 7. 整局結構與節奏

- 整局限時(2~4 分鐘,「本班剩餘時間」);時間到=完成組數多者獲勝、老闆回來結算。
- 0~20s 理解(老闆開場+AI 示範)→ 20~60s 競速 → 60~120s 混亂升級(第一次能量技能)→ 最後 20% 高潮(讀法 B 結局)。
- 理想結局:「玩家不是因為被打死而輸,而是因為對方趁他混亂時完成最後一件工作。」

## 8. AI 的四個角色(繼承 v1)

示範者(首局演一次完整分類)/競爭者(**70% 工作、20% 看局勢干擾、10% 搞笑失誤**)/壓力來源(需求、路線、進度、能量全部可讀,危險可預測)/喜劇角色(拿錯尷尬停頓、被吐瓶、能量滿冒煙、被回收後從管道飛回、快下班露得意表情)。

## 9. UI 原則

必顯示:雙方下班進度(2/3 vs 1/3)+ 每人一條事故能量 + 公開序列。不顯示:第二條怒氣/回收點數/複雜冷卻表。**一條決定勝負,一條決定能力。**

## 10. MVP 數值(v1 §13 + 對齊現有常數)

```text
每組序列:4 元素;獲勝:3 組;單局約 12 件;單局 2~4 分鐘
正確投入 +10~15%、完成組 +10%、被命中 +20~30%
踉蹌(免費拳):0.5s(現 FUMBLE_T)  推開:0.45s(現 PUSH_STAGGER)
擊暈:1.5~2.5s(現 STUN_T 1.2 → 拉到 1.5+)
受擊保護:1~1.5s(現 RESTUN_IMMUNE 0.6 → 拉長)
中央回收停工:≈3s(現 PERFORM_T 2.1/2.6/3.6;吐回版收在 3s 內)
```

## 11. 驗證假設與鎖死規則(繼承 v1 §16/§18)

五假設:看得懂(30s 內 80%)/會注意對手/能量滿不一定立刻用(=有決策)/打架不能取代工作/願意再玩。
鎖死:工作=唯一勝利、分類=主要操作、能量來自工作與受擊、干擾只造時間差、被打者得反擊資源、回收口=招牌干擾演出、勝者提前下班。數據不好先調參數,不推翻核心。

## 13. 待決事項(2026-07-13 全數定案)

1. **垃圾元素表=維持火/冰/毒/電**:毒綠 vs 冰青可辨,水藍會撞色;廢料語義毒比水合理;水的身分留給重錘工具。
2. **吐回出口=固定北牆垃圾管道**:與收容演出「壓縮北送」同一條管線敘事;固定=可預測;離艙有距離=3s 停工自帶路程成本;落地帶 1~1.5s 受擊保護防蹲點;LED「分類結果:可回收性不足,退回」。
3. **供料模型定案(全進 ?tune=1)**:瓶位 6 個、每型在場上限 2、respawn 4~6s、換型加權 50% 玩家需求/30% AI 需求/20% 隨機、保底=需求型場上歸零強制下一個出它;稀缺窗=respawn 延遲本身(3~6s)。
4. **擊暈觸發=能量達標後「三連擊第三拳」才暈,暈成功消耗整條能量**(落空不耗);未達標的第三拳=踉蹌+掉瓶。不做獨立蓄力拳(新輸入+手機按鈕不夠)。
5. **老闆結算 MVP 三段**:開場 HUD 頭像+「給我好好工作!」/下班鐘響頭像回歸宣布/輸家儀式二型——在艙內=封艙壓縮(perform 現成)、不在艙內=加班 gag(垃圾堆+LED「加班中」),2~3s 快速下一局。強制抓去回收的演出 MVP 不做。
6. **舉防保留原樣;精準格擋降級**:黃金窗=強推開+自己 +10% 能量(技巧獎勵保留、不繞過能量閘的反暈);**扛瓶時不能舉防**(雙手滿=搬運者純獵物,攔截戲成立)。
