# v2 規格 — 元素 / 融合 / 元素取得（環境處決 PvP）

> 對應：v2《環境處決 PvP》§3（複雜度搬軸）、§4（三層傷害）、§0.5（卡片＝動詞非 DPS）、§12（速查尺）。
> 一句話結論：**法術融合矩陣砍掉；「融合」搬到地板上變成環境反應。元素是「空間動詞」不是 DPS。**

---

## 1. 核心決定

| 項目 | v1（單機） | v2（PvP） |
|---|---|---|
| 法術數量 | 16（5 單 + 10 融合 + 中性）| **壓到 ~5–6 個可讀動詞** |
| 「融合」在哪 | 兩元素在**法術欄**融合成子彈（土+火→熔岩彈）| **在地板上**：你的元素 × 地形 × 別人丟的元素 → 反應 |
| 元素的本質 | 改造飛彈 / 疊 DPS | **改變對方所處空間的動詞** |
| 玩家怎麼取得 | 每波三選一升級卡 | MVP：**不選，對稱同一套**；變化層：地圖撿元素（之後測）|

> **可讀性是 PvP 的硬天花板**：對手要一眼看懂你能對他的空間做什麼壞事。16 種融合彈無法秒讀 → 砍。

---

## 2. 融合沒死，是搬家

- v1：`土+火 → 熔岩彈`（看不懂的法術書）。
- v2：`土/油 鋪地 + 有人點火 → 火海`（讀地板就懂的危險）。

5 元素兩兩 = **10 種反應仍在**，但從「法術 build」變成「**場上化學反應**」：有作者（誰點的火）、有自作自受（自己踩進去）、有延遲引爆（先鋪後觸發）。組合豐富度保留，而且**更好讀、更好笑**。

> 你不用讀對手的子彈，你讀**地板**。

---

## 3. 元素＝空間動詞（每個都要通過這把尺）

每個元素只問一句：**「它讓我能對別人的空間做什麼新的壞事？」**（不是「我變多強」。）

| 元素 | 空間動詞（對別人的空間做的壞事）|
|---|---|
| 🔥 火 | 點燃你腳下的油/草 → 火海擋路、燒人 |
| ❄️ 冰 | 把你的退路凍成滑冰場 → 滑進空洞/危險區；冰牆封路 |
| ⚡ 雷 | 把水充電 → 你踩進去自電硬直 |
| ☠️ 毒 | 在必經之路留毒區 → 慢性扣血＝慌張計時器 |
| 🪨 土 | 破牆/造牆裂地 → 即時重塑瓶頸、封門 |
| 🌀 風（推）| 把你**送進**以上任何一個（吹進火海/電水/空洞、撞牆）|

**鐵則：沒有一個是「純扣血子彈」。** 純直擊傷害降級為佐料（v2 §4 三層）：環境＝主力擊殺、控場（吹/推/封/緩）＝把人送進環境、直擊＝刻意不足以當主要擊殺。

---

## 4. 玩家怎麼取得元素（三模型 + 決定）

| 模型 | 機制 | 取捨 |
|---|---|---|
| **① 對稱固定 kit（MVP 採用）** | 不選，所有人同一套 ~5–6 動詞 | 最可讀、最對稱、對陌生人最友善、**零 build 陷阱**；= 已驗證的原型形態。變化性來自地圖+創意 |
| **② 開局選 1 招牌** | 進場挑 1 主元素（顏色標示），其餘共用 | 有身份、仍可讀；但輕度不對稱、最接近 build-pull → 暫不採用 |
| **③ 地圖撿元素（之後測的變化層）** | 元素是地圖上**可搶的拾取物**，撿了才有該動詞 | 本身就是聚人爭奪的焦點（衝同點互炸）、動態；= v2 講的「卡片」。風險：可讀性靠強視覺、別跟「搶結晶」搶焦點 |

**決定：**
1. **MVP 用 ①**（對稱固定 kit）——連選單都不用做，把力氣全押在把死亡做好笑（規格 A）。
2. 好笑驗穩後，把 **③** 當變化/爭奪層測一版。
3. **② 暫不做**（可讀紅利最小、最接近要離開的 build 軸）。

---

## 5. ⚠️ Build 陷阱警報（v2 §12 第 7 題）

只要「選元素 / 撿元素」開始變成「疊 DPS、解鎖更強組合、養成深度」，就是 **Path A（build 失控爽感）在把你拉回去**。

- 元素卡 = **新動詞**，不是更高傷害。
- 每張卡刷尺：改變空間（放大）vs 純扣血（降級）。
- 反例（別做）：「火 II：火球傷害 +30%」。正例（要做）：「火：點燃油/草成火海」。

---

## 6. 對短答的定稿

- **「融合還在嗎（土+火）？」** → 法術融合矩陣砍掉；**融合搬到地板上 = 環境反應**（土/油 + 火 = 火海）。組合豐富度保留，更好讀更好笑。
- **「玩家怎麼選元素？」** → MVP **不選、對稱同一套小詞彙**；變化性之後用**地圖撿元素**（可搶焦點）加，別做成隱藏 build。

---

## 7. 紀律提醒

這套**別在「3–4 人笑出聲」（規格 A 的驗收）之前過度工程**。對稱固定 kit 不需要任何選擇 UI；元素取得系統等 fun 驗穩、要加變化時再上 ③。先驗 fun → 再驗 netcode → 才提交完整棧。
