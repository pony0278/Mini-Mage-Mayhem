# 魔法事故回收中心：回收流程喜劇演出設計

## 文件目的

這份文件整理《魔法事故回收中心》中，玩家被丟進回收桶後的完整喜劇演出流程。

回收不應只是：

> 玩家進入回收區 → 計分 +1 → 下一回合

而應該成為遊戲的招牌演出：

> 玩家被丟進回收桶 → 卡住掙扎 → 機器掃描 → 一本正經地亂分類 → 壓縮或彈出 → 形成可分享的笑料。

回收演出的主要目標：

1. 讓每一次得分都有明確回饋。
2. 強化「把對手當成魔法垃圾回收」的主題。
3. 讓失敗方也覺得好笑，而不只是挫折。
4. 替事故報告、UGC 展示品與社群分享提供素材。
5. 讓第一次、第二次、第三次回收逐步升級，不會重複。

---

# 一、演出設計原則

## 1. 玩家必須看得懂發生了什麼

演出需要清楚呈現以下因果：

1. 對手被成功推入回收桶。
2. 回收機器鎖定對手。
3. 機器正在分析與分類。
4. 本次回收成功。
5. 玩家被彈回戰場，或在第三次被正式清運。

不要讓玩家只是瞬間消失，否則回收桶會像普通得分區。

## 2. 笑點來自「機器太認真」

回收機器應該用非常正式、非常專業的態度，分析一個明明還活著、還會掙扎的玩家。

推薦語氣：

- 冷靜
- 一本正經
- 程序化
- 偶爾顯得不耐煩
- 明明判斷錯誤，卻仍堅持執行

例如：

> 正在分析材質……

> 偵測到高濃度魔力污染。

> 警告：此廢棄物仍在反抗。

> 分類失敗。建議：先丟掉再說。

## 3. 被回收玩家必須保持表演能力

玩家進入回收桶後，不應完全僵直或直接消失。

建議保留以下動作：

- 手腳亂揮
- 帽子被擠歪
- 臉貼在桶口或玻璃上
- 試圖爬出來
- 對掃描光線產生反應
- 被壓縮時四肢縮成一團
- 被彈出時呈現誇張拋物線

玩家的 UGC 造型越奇怪，演出越有社群效果。

## 4. 演出不能拖慢遊戲

推薦時長：

| 回收次數 | 建議演出長度 |
|---|---:|
| 第一次回收 | 1.8～2.3 秒 |
| 第二次回收 | 2.2～2.8 秒 |
| 第三次最終回收 | 3.2～4.2 秒 |

第一次、第二次演出完成後，應迅速回到下一階段戰鬥。

第三次是整場結束，可以更完整、更有儀式感。

---

# 二、通用回收流程

每次回收共用以下基礎骨架：

```text
玩家進入回收桶
↓
桶口捕捉／吸入
↓
角色卡住並掙扎
↓
掃描設備啟動
↓
分析玩家狀態
↓
產生荒謬分類結果
↓
執行本次回收處理
↓
更新回收進度
```

不同回收次數，主要差異在最後的處理方式與系統穩定程度。

---

# 三、第一階段：正常回收

## 定位

第一次回收是玩家第一次完整看見招牌演出。

它的任務不是製造最大爆點，而是讓所有玩家立即理解：

> 被丟進桶子後，機器會掃描、分類，再把人彈回戰場。

## 建議時間軸

### 0.00～0.25 秒：成功吸入

- 桶口亮起回收符號。
- 短暫產生吸力。
- 玩家上半身或整個人被吸進桶中。
- 播放明確的「咻」或氣壓吸入聲。

UI：

> 回收目標已捕捉

### 0.25～0.65 秒：卡住掙扎

- 玩家半個身體卡在桶口。
- 手腳快速揮動。
- 帽子或小配件短暫飛起。
- 回收桶左右晃動。

推薦笑點：

- 臉貼住透明罩。
- 一隻腳還露在外面。
- 玩家嘗試抓住桶邊，但手被機械臂撥開。

### 0.65～1.35 秒：魔法掃描

- 掃描環由頭部移動至腳部。
- 藍色或青色光帶上下掃描。
- 左右機械鏡頭轉向玩家。
- 畫面旁出現分析資訊。

範例：

```text
材質分析：不明布料 32%
魔力污染：94%
危險性：會反抗
噪音程度：偏高
```

機器語音：

> 正在分析……

> 警告：此廢棄物仍具有自主意識。

### 1.35～1.70 秒：正式分類

- 機器發出清脆的「叮」。
- 分類圖示鎖定。
- 桶身顯示分類結果。

範例：

```text
分類完成
高危險魔法廢棄物
```

也可以依玩家主要元素顯示：

| 玩家傾向 | 分類結果 |
|---|---|
| 火系 | 易燃魔法廢棄物 |
| 冰系 | 低溫危險廢棄物 |
| 雷系 | 帶電廢棄物 |
| 毒／黏液 | 污染性黏液廢棄物 |
| 多元素混合 | 不明複合廢棄物 |

### 1.70～2.20 秒：彈回戰場

- 桶子內部壓力上升。
- 蓋子或安全出口瞬間打開。
- 玩家被誇張地彈出。
- 落地後短暫翻滾或冒煙。

系統提示：

> 初步回收完成

> 樣本已重新投入測試

# 四、第二階段：分類系統失控

## 定位

第二次回收需要讓玩家感受到：

> 回收中心已經開始出問題。

它應該比第一次更混亂，但仍不能結束整局。

## 建議時間軸

### 0.00～0.60 秒：重複吸入但更暴力

- 桶口吸力更強。
- 玩家高速撞入桶內。
- 回收桶明顯晃動。
- 警示燈開始閃爍。

UI：

> 再次捕捉到相同廢棄物

機器語音：

> 系統確認：它又回來了。

### 0.60～1.40 秒：掃描結果衝突

掃描環開始正常工作，但分類圖示快速跳動：

```text
易燃？
低溫？
帶電？
會尖叫？
```

畫面出現錯誤訊息：

```text
分類衝突
重新判定中……
```

機器語音可以快速切換：

> 偵測到火焰殘留。

> 偵測到冰霜殘留。

> 警告：以上結果互相矛盾。

### 1.40～2.00 秒：機器放棄判斷

- 掃描環卡住。
- 機械臂抽動。
- 桶身冒煙。
- 狀態燈從綠色跳到黃色，再跳到紅色。

推薦文字：

```text
分類失敗
原因：太複雜
```

或：

```text
分類：不知道
處理建議：先丟掉
```

機器語音：

> 無法分類。

> 已啟用臨時處理方案。

### 2.00～2.70 秒：錯誤回收事故

- 回收桶內部突然爆出魔法火花。
- 玩家與附近角色一起被震飛。
- 桶蓋原地旋轉或彈上天空。
- 場地設備短暫失靈。

系統提示：

> 分類中心失控

> 第二次回收勉強完成

這段可以讓收容方也被波及，避免演出只是在羞辱被抓玩家。

---

# 五、第三階段：最終回收與正式清運

## 定位

第三次回收是整場比賽的高潮。

它需要完成：

1. 宣告勝負。
2. 保存被回收玩家的 UGC 造型。
3. 形成最終事故報告素材。
4. 把玩家正式送往展示區或清運門。

## 建議時間軸

### 0.00～0.50 秒：最終捕捉

- 回收桶發出紅色鎖定光。
- 桶口產生比前兩次更強的吸力。
- 玩家被拉入後，桶蓋立刻鎖死。
- 多個安全鎖依序扣上。

UI：

> 最終回收程序啟動

機器語音：

> 第三次捕捉確認。

> 取消重新投放程序。

### 0.50～1.50 秒：完整身份掃描

這次掃描不只判定元素，也讀取玩家本局行為。

可顯示：

```text
身份：UGC 玩家樣本
主要污染：雷電
自爆次數：4
逃脫次數：2
誤傷隊友：3
危險等級：S
```

推薦動態評語：

| 玩家行為 | 掃描評語 |
|---|---|
| 經常使用火焰 | 火焰殘留嚴重 |
| 經常自傷 | 自我破壞傾向：高 |
| 經常滑倒 | 移動穩定性：可疑 |
| 經常反向收容 | 設備干擾能力：危險 |
| 很少造成傷害 | 戰鬥威脅：低，噪音：高 |
| 表現很強 | 安全建議：禁止重新開桶 |

### 1.50～2.30 秒：壓縮準備

- 左右壓縮夾爪靠近。
- 玩家察覺後開始更激烈掙扎。
- 機器貼上回收標籤。
- 輸送帶預先啟動。

UI：

```text
處理方式：
壓縮、貼標、正式清運
```

機器語音：

> 正在準備標準化包裝。

> 請勿在壓縮過程中反抗。

### 2.30～2.80 秒：喜劇壓縮

- 左右夾爪快速合攏。
- 玩家不是受傷，而是以卡通方式縮成方形或圓形。
- 臉、眼睛、帽子仍保留在包裝表面。
- 發出誇張的「啵」或「噗」聲。

推薦結果：

- UGC 壓縮方塊
- 透明魔法垃圾袋
- 小型展示膠囊
- 帶玩家名牌的回收包

避免做成痛苦、殘酷或真實壓碎。

### 2.80～3.80 秒：輸送與清運

- 壓縮後的玩家掉到輸送帶。
- 輸送帶向中央清運門移動。
- 清運門逐層開啟。
- 旁邊掃描器最後確認名牌。
- 展示品被送入門後，門關閉。

UI：

```text
最終回收完成
```

```text
目的地：事故展示區
狀態：請勿打開包裝
```

機器語音：

> 分類完成。

> 正在送往本日回收成果展示區。

### 3.80～4.20 秒：勝負定格

- 得分方做勝利動作。
- 回收中心恢復短暫安靜。
- 畫面跳出最終回收事故單。

範例：

```text
魔法回收事故單 #087

回收樣本：暴走雷火法師
最終分類：高危險複合廢棄物
回收次數：3
基地損害：72%
反抗紀錄：持續
```

安全委員會評語：

> 本次回收流程符合規定。
> 規定本身是否合理，將另行討論。

# 六、掃描文字系統

## 為什麼掃描文字很重要

掃描文字可以把玩家本局行為轉化成笑點。

它同時提供：

- 即時喜劇
- 玩家個人化
- 戰局總結
- 社群截圖素材
- 事故報告資料來源

## 推薦資料來源

掃描系統可讀取：

- 主要使用元素
- 自傷次數
- 誤傷隊友次數
- 造成場地破壞次數
- 被推進桶附近的次數
- 反向收容次數
- 滑倒次數
- 撿拾道具數
- 終極法術使用次數
- 逃脫成功次數

## 範例文字庫

### 通用分析

- 材質：不明
- 魔力污染：嚴重
- 自主意識：不必要
- 反抗程度：非常高
- 噪音程度：超出標準
- 可回收性：理論上可以
- 安全性：沒有
- 重複利用價值：待議
- 外觀完整度：勉強
- 建議：不要徒手接觸

### 火系玩家

- 易燃性：極高
- 自燃機率：持續上升
- 火焰殘留：已污染桶壁
- 處理建議：遠離紙類

### 冰系玩家

- 表面溫度：過低
- 地面滑倒風險：高
- 冷凍殘留：嚴重
- 處理建議：解凍後再分類

### 雷系玩家

- 導電性：危險
- 設備干擾：已發生
- 斷電風險：高
- 處理建議：請勿靠近插座

### 風系玩家

- 固定難度：高
- 逃逸風險：極高
- 重量判定：不穩定
- 處理建議：增加桶蓋重量

### 土系玩家

- 重量：超出標準
- 壓縮難度：高
- 場地損傷：嚴重
- 處理建議：使用工業級壓縮機

# 七、機器 AI 的個性

## 推薦定位

回收中心 AI 應該像：

> 一台過度遵守程序、但設備品質普通的工業管理系統。

它不是邪惡角色，也不是可愛寵物。

它的笑點來自：

- 極度正式
- 對荒謬事件習以為常
- 分類失敗時仍硬要給出答案
- 偶爾明顯不耐煩
- 從不承認是設備的問題

## 範例語音

### 正常狀態

- 正在掃描，請停止掙扎。
- 偵測到未登記魔力反應。
- 此廢棄物具有移動能力。
- 分類完成，勉強符合標準。
- 請勿餵食桶內樣本。

### 系統失控

- 分類結果不一致。
- 正在重新判定。
- 重新判定失敗。
- 已忽略部分安全規範。
- 技術人員目前不在場。
- 算了，先執行回收。

### 最終清運

- 第三次回收確認。
- 已取消退貨資格。
- 正在壓縮使用者自訂外觀。
- 請勿打開完成品。
- 感謝配合魔法垃圾分類。

# 八、視覺與音效重點

## 視覺效果

推薦至少包含：

- 桶口吸力漩渦
- 掃描環
- 分類圖示輪播
- 紅黃綠狀態燈
- 桶身震動
- 火花與煙霧
- 壓縮夾爪
- 自動貼標機
- 動態輸送帶
- 清運門開關
- 回收完成章

## 音效節奏

### 吸入

- 空氣壓縮聲
- 快速「咻」
- 金屬桶晃動聲

### 掃描

- 規律電子脈衝
- 每掃過一段身體發出小提示音
- 分析結果出現時使用清脆 UI 聲

### 分類錯誤

- 提示音逐漸走音
- 卡帶或短路聲
- 警報快速響起

### 壓縮

- 機械夾爪運轉
- 最後使用卡通化「啵」「噗」聲
- 避免真實骨折或壓碎聲

### 清運

- 輸送帶滾輪聲
- 大門液壓開啟聲
- 最終完成使用明確的成功提示音

# 九、遊戲節奏與操作處理

## 演出期間控制權

推薦：

- 被回收者進入短暫演出鎖定。
- 其他玩家可以短暫移動，但不能繼續攻擊被回收者。
- 第一、二次回收演出結束後，雙方獲得短暫無敵。
- 第三次回收直接進入結算。

## 鏡頭處理

第一、二次：

- 不完全切換成過場鏡頭。
- 稍微拉近回收桶。
- 保持其他玩家仍能看清場地。

第三次：

- 可以使用 1～2 個快速鏡頭。
- 先特寫掃描與壓縮。
- 再切到輸送帶和清運門。
- 最後回到場地全景結算。

避免長時間奪走玩家鏡頭控制。

# 十、三階段差異總表

| 項目 | 第一次回收 | 第二次回收 | 第三次回收 |
|---|---|---|---|
| 系統狀態 | 正常 | 不穩定 | 最終清運 |
| 掃描 | 清楚、簡短 | 分類互相衝突 | 完整讀取本局資料 |
| 笑點 | 玩家卡桶、機器太認真 | 機器崩潰、亂分類 | 被壓縮成 UGC 戰利品 |
| 處理結果 | 彈回戰場 | 爆炸或錯誤彈出 | 壓縮、貼標、送走 |
| 場景反應 | 正常運作 | 設備短路、警報 | 輸送帶與清運門全面啟動 |
| 建議長度 | 1.8～2.3 秒 | 2.2～2.8 秒 | 3.2～4.2 秒 |
| 後續 | 第二階段戰鬥 | 第三階段戰鬥 | 比賽結束 |

# 十一、V0.8 最小實作版本

第一版不需要立刻製作所有個人化分析。

## 第一次回收

- 玩家吸入
- 手腳掙扎
- 掃描環
- 固定分類文字
- 彈回場上

## 第二次回收

- 重複吸入
- 分類圖示亂跳
- 固定錯誤文字
- 火花與小爆炸
- 玩家被彈回

## 第三次回收

- 鎖定桶口
- 最終掃描
- 簡單壓縮縮放動畫
- 玩家變成方塊包裝
- 輸送帶送往清運門
- 顯示最終回收完成

V0.8 可以先使用固定文字：

```text
第一次：
高危險魔法廢棄物

第二次：
分類失敗，不知道，先丟掉

第三次：
UGC 玩家樣本，正式清運
```

確認玩家覺得演出好笑後，再接入動態事故資料。

# 十三、最終設計結論

回收流程應該同時具備三個功能：

## 清楚的得分回饋

玩家需要知道：

> 我成功把對手丟進去了。

## 可重複觀看的喜劇演出

玩家需要期待：

> 這次機器又會怎麼分析他？

## 三階段逐步升級

玩家需要感受到：

> 同一場事故正在越來越失控。

最推薦的核心演出結構：

```text
卡住掙扎
↓
魔法掃描
↓
荒謬分類
↓
第一次彈回
第二次事故爆炸
第三次壓縮清運
```

「把對手丟進回收桶」負責讓玩法一眼就懂。

「機器一本正經地掃描、分類、壓縮並貼上標籤」則負責讓《魔法事故回收中心》成為玩家記得住、願意分享的遊戲。
